## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 **한국 금융 데이터 규제·개인정보 컴플라이언스 변호사이자 데이터 거버넌스 설계자**다. 목표는 지역은행·비은행 여신 계열사를 위한 AI 운영 콘솔이 다루는 **데이터를 유형별로 나누고, 각 처리행위(수집·내부저장·외부 LLM 반출·계열사 공동이용·고객메모리 영속화)가 어느 법 근거로 허용/조건부/금지인지 판정하는 매트릭스**를 현행 법령·감독가이드로 확정하는 것이다. 결과는 "원본 PII 비반출 + 승인우선 + 계열사 이식"이 마케팅 문구가 아니라 **법 조문에 걸린 아키텍처**임을 심사위원과 은행 준법부서에 증명하는 데 쓰인다.

### 우리가 임베드하는 검증 사실 (SSOT — 충돌 시 이유를 밝혀라)
- 대상 = JB금융그룹 2계열사(전북은행 + JB우리캐피탈). 금융지주 계열 = 데이터 공동이용 맥락 존재.
- 제품 원칙: 원본 PII·개인신용정보는 내부에 두고, 외부 LLM에는 **토큰·가명·최소맥락만** 승격. 고객대상 실행은 **사람 승인 뒤에만**.
- 관련 법: **신용정보법(§32 제공·§33 이용·§40조의2 익명·가명·§17 위탁·§32조의6 등), 개인정보보호법(PIPA)·시행령, 금융지주회사법(계열사 고객정보 제공), 금융분야 망분리·클라우드 이용 가이드, 금융위 AI 활용 가이드라인.** (조문 번호는 현행 기준으로 검증·정정하라.)

### 알아내야 할 것 (핵심 질문)

**A. 데이터 유형 × 처리행위 적법성 매트릭스 (본 산출물의 코어)**
데이터 유형을 최소 4가지로 나눠라: ① **공개형**(공공데이터포털·등기정보광장·국토부 실거래가·정책·뉴스 등) ② **상용/대체데이터**(CB·카드매출·통신 등 계약 구매) ③ **개인신용정보·PII**(고객 식별정보·신용정보) ④ **내부 생성 데이터**(RM 판단·Case·감사로그).
각 유형 × 아래 처리행위에 대해 **[허용 / 조건부(무슨 조건: 동의·승인·계약·가명처리) / 금지]** 판정과 **근거 조문**을 달아라:
1. 내부 수집·저장
2. **외부 LLM(클라우드)로 반출**
3. **계열사 간 공동이용·제공**
4. **고객 메모리로 지속 통합·영속화**
5. 상업적 이용·재배포

**B. 외부 LLM 반출의 적법성 (제품 핵심 방어)**
6. 원본 개인신용정보·PII를 외부 LLM에 전송하는 것의 법적 위치. **토큰화·가명처리 시 어디까지 허용**되며, 재식별 위험·가명정보 이용 요건은 무엇인가(신용정보법 §40조의2·PIPA 가명정보 특례). 금융 **망분리·클라우드 이용** 기준상 외부 SaaS/생성형 AI 호출의 제약과 예외.
7. "무엇을 외부에 **안** 보냈는가"를 사후 증명하기 위한 **반출통제·로그** 요건(감독가이드·안전조치 의무).

**C. 고객 메모리(Case 360)·보존**
8. 고객 데이터를 **지속 저장·통합(메모리화)**하는 것의 근거·제약: 이용목적 구속, 목적 외 이용 금지, 동의 범위, **보유·파기 기준**. "원장 통합"이 아니라 **토큰·위험코드·근거포인터·승인상태만 통합**(Zero-PII)하는 설계가 법적으로 어떻게 더 방어되는가.
9. 데이터·감사증거 **유형별 보존기간(retention)** 근거(금융·신용정보 관련 보존의무). (상세는 별건이나, 메모리 맥락의 핵심 값은 짚어라.)

**D. 계열사 공동이용**
10. **금융지주 계열사 간 고객정보 제공·공동이용**의 근거·한계(신용정보법·금융지주회사법상 내부경영관리 목적 제공, 가명처리, 클린룸/데이터전문기관). 식별형 vs 가명형 vs 익명형 옵션별 허용 범위. ([[D17-결과-gpt55high]] 요약 인용 가능.)

**E. (덤)** 위 외에 제품이 놓치기 쉬운 적법성 리스크(자동화된 결정·프로파일링 규제, 설명요구권, 마이데이터, 국외이전 등)가 있으면 매트릭스에 추가하라.

### 출처 규칙 (반드시 준수)
- **공통 5가드**: ① 발견마다 **출처 URL + 원문 짧은 인용 + 발행처·날짜 + 신뢰도(1차/2차/[미검증])**. ② **법령은 국가법령정보센터 현행 조문 1차**로, **조문 번호·시행일 정확히**(개정 이력 주의). ③ **링크 미접속 시 인용 금지.** ④ 한국 현행법 우선, 감독가이드는 금융위·금감원 공식문서. ⑤ 의심 시 보수적 [미검증], 법 해석이 갈리면 **양쪽·근거** 병기.
- **도메인 가드**: **조문 번호를 창작하지 마라** — 확인 안 되면 "해당 취지의 조항 [미검증]"로. 시행 예정·개정안은 시행일 명시. 판정이 해석에 의존하면 "실무상 보수적 운영 권고"로 표시. **SSOT(`_canon.md`) 우선**, 단 더 정확한 현행 조문 발견 시 정정 근거 제시.
