# 연결-2 — 인텔리전스 — 통합 고객지능·정책룰화·시뮬레이션(합법범위는 D17/D5a 분리)

## 연결된 발견
- 정책 문서는 안내문이 아니다. 스트레스 DSR, 전세사기 피해자 지원, 소상공인 이자환급, 보이스피싱 대응은 모두 시행일·대상·한도·예외·의무로 쪼개져 룰엔진 변수로 바뀐다. 여기에 RTMS, HUG, 국세청, 상권정보, 행안부 인허가를 붙이면 “정책 → 고객/물건/업종 매칭 → 조치 후보”가 된다. [[D21-결과.md]] [[D7a-결과.md]] [[D7a-결과-2.md]]
- 고객지능의 핵심은 Customer 360 자체가 아니라 “원본을 합치지 않고도 같은 Case를 설명하는 방법”이다. D21은 entity resolution과 고객 메모리의 효용을 말하고, D17은 그룹 공통 Hub에 `subject_token`, `risk_code`, `score_band`, `evidence_pointer`, `approval_state`만 올리라고 선을 긋는다. 통합 고객지능은 원장 통합이 아니라 토큰·근거·승인 상태의 통합이다. [[D21-결과.md]] [[D17-결과-gpt55high.md]]
- D11의 Case → AgentRun → Skill → Evidence → Approval → Audit 구조가 D17의 클린룸 운영모델과 맞물린다. Evidence는 원문 복제가 아니라 provenance 포인터가 되고, Approval은 CBR의 Revise 단계가 되며, Audit은 Retain과 change history가 된다. 이 구조가 있어야 시뮬레이션 결과도 “추천”이 아니라 감사 가능한 판단 초안이 된다. [[D11-결과.md]] [[D17-결과-gpt55high.md]]
- 데이터 조달 레이어는 3층이다. 공개형 API는 기본 경로, 인터넷등기소는 유료·건별·사람 승인형 보강 경로, 금융결제원·CB·카드·더치트는 계약형 확장 경로다. 이 구분을 제품에 박으면 과장 없이 “즉시 MVP”와 “제휴 후 확장”을 분리할 수 있다. [[D7a-결과.md]] [[D7a-결과-2.md]] [[D7b-결과.md]]
- simulation-before-action은 금융 스트레스 테스트와 Agent 메모리 구조를 잇는 다리다. D21은 시나리오·몬테카를로·디지털 트윈·월드모델을 행동 전 검증 패턴으로 제시하고, D11은 실행 결과와 교훈을 다음 Case에 남기는 구조를 준다. 즉 LocalGuard OS의 시뮬레이션은 예측 엔진이 아니라 “승인 전 실패 탐지 장치”다. [[D21-결과.md]] [[D11-결과.md]]

## 1차 인사이트
- 인텔리전스의 차별점은 “더 많은 데이터”가 아니라 “정책·고객·외부사건·시뮬레이션을 같은 Case ID 아래 묶는 운영계약”이다. 정책룰은 Skill이 되고, 고객·물건·업종 신호는 Evidence가 되고, 시뮬레이션은 Approval 전 AgentRun으로 남는다. [[D21-결과.md]] [[D11-결과.md]]
- JB LocalGuard OS는 Customer 360을 주장하면 위험하고, “Zero-PII Case 360”을 주장하면 강해진다. 전북은행과 JB우리캐피탈의 원장은 각자 내부에 두고, 공통 레이어에는 토큰·도메인·위험코드·근거 포인터·승인상태만 올린다. 이게 하드룰 PII 비반출과 멀티도메인을 동시에 만족한다. [[D17-결과-gpt55high.md]] [[D17-결과-geminiflash.md]]
- 정책룰화는 RM 생산성 기능이 아니라 내부통제 기능이다. 금융위 보도자료의 시행일·대상·예외조건을 룰로 만들고, 룰 변경 로그와 사람 승인 로그를 남기면 “AI가 판단했다”가 아니라 “정책 변경을 감사 가능한 절차로 반영했다”가 된다. [[D21-결과.md]] [[D11-결과.md]]
- 전세·소상공인·사기 도메인은 한 화면에 묶어도, 데이터 레인은 나눠야 한다. RTMS/HUG/국세청/상권정보는 공개형 기본 신호, 등기·CB·카드·더치트는 승인형 보강 신호, 금융결제원·ASAP류는 내부망 또는 계약형 신호다. 이 레인 구분이 곧 제품의 신뢰성이다. [[D7a-결과.md]] [[D7a-결과-2.md]] [[D7b-결과.md]]
- 심사위원에게는 “시뮬레이션이 맞춘다”가 아니라 “시뮬레이션이 막는다”로 말해야 한다. 조치 초안은 외부 LLM이 도와도 고객 연락·한도 차단·상품 권유는 인간 승인 전까지 Hold다. 이 문장이 금융 AI 가이드라인의 보조수단 원칙과 바로 맞는다. [[D21-결과.md]] [[D17-결과-gpt55high.md]]

## 모순·텐션
- D17 gemini는 전북은행·광주은행·JB우리캐피탈 3계열사를 전제로 쓴다. 현재 제품 정의는 JB 2계열사=전북은행+JB우리캐피탈·RM에이전트다. reconcile: 본선 제출 본문은 2계열사로 고정하고, 광주은행은 “확장 가능한 동일 지주 패턴”으로만 부록 처리한다. [[D17-결과-geminiflash.md]]
- D7a 두 판본에서 등기정보광장 API 키 유효기간이 1년과 24개월로 엇갈린다. reconcile: canon에는 확정 수치로 올리지 말고 “공식 재확인 필요”로 둔다. 전세 권리분석은 어차피 개별 등기사항증명서 대량 API가 확인되지 않았으므로 사람 승인형 유상 보강 절차가 맞다. [[D7a-결과.md]] [[D7a-결과-2.md]]
- D7b는 공공 데이터 기본 경로를 “반출 및 가공 제약이 없음”에 가깝게 쓴다. D7a는 data.go.kr의 “이용허락범위 제한 없음”에도 제3자 권리 예외가 있다고 경고한다. reconcile: 제품 문구는 “공개형 저마찰 데이터 우선, 원문 메타·제3자 권리 예외 확인”으로 낮춘다. [[D7b-결과.md]] [[D7a-결과.md]]
- D21은 Customer 360과 엔티티 해소의 가치를 말하지만, D17은 계열사 원장 통합을 강하게 제한한다. reconcile: “통합 고객지능”은 법적 통합 원장이 아니라 Case Hub의 토큰 기반 통합 관측으로 정의한다. 합법 범위 판정은 D17/D5a에 남긴다. [[D21-결과.md]] [[D17-결과-gpt55high.md]]

## 제품·심사 매핑
- 1=주제: 지역은행 RM이 정책·공공데이터·계열사 위험신호를 놓치지 않게 하는 인텔리전스 OS로 잡는다. “AI 상담도구”가 아니라 지역 금융 리스크 운영 레이어다. [[D21-결과.md]]
- 2=금융가치: 소상공인은 국세청 상태조회·상권정보·카드매출 보강, 전세는 RTMS·HUG·등기 승인조회, 사기는 경찰청/KISA·더치트·ASAP 내부망 신호로 조기경보를 만든다. 공개형과 계약형을 나눠 실전 도입 가능성을 증명한다. [[D7a-결과.md]] [[D7b-결과.md]]
- 3=Agent설계: Case → AgentRun → Agent → Skill → Evidence → Approval → Audit이 CBR 4R, provenance, episodic memory와 대응한다. 이 구조가 Agent를 “자동 실행자”가 아니라 “근거 있는 초안 작성자”로 제한한다. [[D11-결과.md]]
- 4=MVP검증: 합성·공개·익명/집계·가명 데이터 순으로 PoC를 돌리고, 외부 LLM에는 비식별 요약만 보낸다. 데모에서는 원본 PII 없이도 Case 생성, 정책룰 매칭, 시뮬레이션, 승인 Hold, Audit 기록이 끝까지 보여야 한다. [[D17-결과-gpt55high.md]]
- 5=혁신·운영: 혁신은 생성형 AI 자체가 아니라 하드룰 PII 비반출, 승인우선, 정책룰화, 케이스 메모리, 시뮬레이션 전 검증의 결합이다. 금융권이 실제로 받아들일 수 있는 AI 운영모델로 포지셔닝한다. [[D17-결과-geminiflash.md]] [[D21-결과.md]]

## 시연 한 컷
- RM이 “전북은행 소상공인 만기연장 Case”를 연다. 화면 왼쪽에는 국세청 상태조회 30분 갱신 플래그, 상권정보, RTMS/HUG 같은 공개 신호가 Evidence로 붙는다. 가운데에는 정책문서에서 추출된 지원·제외·한도 룰이 Skill로 적용된다. 오른쪽에는 3개 조치안의 시뮬레이션 결과가 뜨지만, 고객 연락과 조건 변경은 `PENDING` 상태다. RM이 근거 포인터와 PII 차단 로그를 확인한 뒤 승인해야만 Action이 열린다. [[D7a-결과-2.md]] [[D11-결과.md]] [[D17-결과-gpt55high.md]]

## canon 갱신 후보
- “통합 고객지능” 정의를 “계열사 원장 통합”이 아니라 “Zero-PII Case Hub 기반 토큰·위험코드·근거포인터·승인상태 통합”으로 고정 제안. [[D17-결과-gpt55high.md]]
- MVP 데이터 레인 추가 제안: 공개형 기본 경로(RTMS/HUG/국세청/상권정보/행안부 인허가), 유료·건별 승인 경로(인터넷등기소), 계약형 확장 경로(금융결제원/CB/카드/더치트). [[D7a-결과.md]] [[D7a-결과-2.md]] [[D7b-결과.md]]
- 제품 메시지 추가 제안: “시뮬레이션은 예측 보장이 아니라 승인 전 실패 탐지 장치.” [[D21-결과.md]]
- 2계열사 범위 canon 확인 필요: D17 gemini의 광주은행 포함 문구는 본선 제출 정본에서는 확장 사례로 격하. [[D17-결과-geminiflash.md]]
