# 연결-3 — 규제·신뢰 — PII 비반출·승인우선의 법·기술 정합

## 연결된 발견
법은 토큰화 자체를 보지 않는다. **추가정보 분리보관, 접근권한 분리, 처리기록 3년 보존, 재식별 시 즉시 처리중지·회수·삭제**를 본다. 기술은 같은 결론으로 간다. 외부 LLM 앞에 **데이터 분류 → PII 탐지 → 토큰화/가명화 → 최소 맥락화 → 외부 호출 → 응답 DLP 스캔 → 정책 검증 → 내부 역치환 → 감사 기록** 브로커가 필요하다. 법과 기술이 같은 구조를 요구한다. [[D5a-결과]] [[D10-결과]]

망분리 정책은 외부 생성형 AI를 완전히 막지 않는다. 2024년 11월 **9개 금융회사 10개 혁신금융서비스**, 2024년 12월 **16개 금융사 25개 서비스** 지정으로 길을 열었다. 2026년 4월 20일부터 내부업무망 SaaS 예외도 일부 정규화됐다. 다만 **고유식별정보나 개인신용정보**는 일반예외 밖이다. 가명정보도 별도 혁신금융서비스 경로가 더 안전하다. [[D5b-결과]] [[D12-결과]]

승인우선은 보안장치가 아니라 운영통제다. 승인 버튼만 있으면 rubber-stamping이 된다. 의미 있는 승인은 **근거 묶음, 이유 코드, 이중승인, 반대·재심·상향보고, SLA, kill switch**와 붙어야 한다. 그래서 JB LocalGuard OS의 `Approval`은 UX 요소가 아니라 규제 방어 레이어다. [[D15-결과]]

감사원장은 “로그 저장소”가 아니다. 전자문서 증거력은 **무결성·동일성**이 핵심이고, 제품은 append-only, 해시/전자서명, 시간동기화, chain-of-custody, forensic export를 가져야 한다. D5a의 3년 가명처리 기록과 D15의 전자금융거래기록 5년 보존은 서로 다른 증거층이다. [[D5a-결과]] [[D15-결과]]

## 1차 인사이트
- **제품의 핵심 주장은 “외부 LLM을 쓴다”가 아니라 “외부 LLM을 법적으로 쓸 수 있는 작업만 라우팅한다”다.** 원본 개인신용정보는 내부에 남기고, 외부에는 가명·토큰화된 최소정보만 보내며, 출력 재식별 위험까지 스캔해야 한다. [[D5a-결과]] [[D5b-결과]] [[D10-결과]]

- **Case→AgentRun→Agent→Skill→Evidence→Approval→Audit은 규제언어로 번역된다.** Case는 업무목적, AgentRun은 처리기록, Skill은 허용된 도구권한, Evidence는 근거·출처, Approval은 인간 개입, Audit은 감독·분쟁 증거다. 이 운영계약 자체가 법·보안·심사 설명의 골격이다. [[D5a-결과]] [[D10-결과]] [[D15-결과]]

- **토큰화의 차별점은 암호기술이 아니라 책임선이다.** 토큰↔원본 키가 은행 내부 HSM/별도 저장소에 있고, 외부 사업자가 재결합할 수 없고, 목적 외 사용이 계약상 금지될 때 비로소 위탁 구조로 방어된다. [[D5a-결과]]

- **승인우선은 false positive까지 설계해야 신뢰가 된다.** ASAP AI 플랫폼은 12주간 **186.5억 원**, 6개월 기준 **474.6억 원** 차단 성과를 냈지만, 지급정지 오차단은 **5영업일 내** 심사결과 통보 같은 구제절차가 필요했다. 승인 콘솔은 차단보다 재심·해제·설명 SLA를 같이 보여줘야 한다. [[D15-결과]]

- **심사위원에게 가장 강한 한 컷은 “차단”이 아니라 “차단된 이유와 해제 가능한 경로”다.** 금융 AI 신뢰는 자동화 성능보다 설명·복구·책임추적에서 갈린다. [[D15-결과]]

## 모순·텐션
- **D12는 “혁신금융서비스 지정을 통해서만” 적법 해결된다고 강하게 쓴다. D5b는 2026년 4월 20일부터 내부업무망 SaaS 예외가 일부 정규화됐다고 본다.** reconcile: 비식별·비신용정보형 내부업무 SaaS는 정규 예외, 가명정보·개인신용정보 연계 외부 생성형 AI는 샌드박스 경로로 나눠 쓴다. [[D5b-결과]] [[D12-결과]]

- **D12의 “고객 정보 유출 가능성이 전무하다”는 표현은 위험하다.** D5a는 외부 LLM 전송이 자동 적법이 아니라고 했고, D10은 프롬프트 인젝션·로그·캐시·준식별자 재식별 위험이 남는다고 했다. reconcile: “전무” 대신 “원본 PII 비반출과 다층통제로 위험을 실질 저감한다”로 고친다. [[D5a-결과]] [[D10-결과]] [[D12-결과]]

- **기록 보존기간이 3년, 5년, 1년 이상으로 갈린다.** reconcile: 가명처리 조치기록은 3년, 전자금융거래기록·접속기록 등은 각 법령별 기간을 별도 retention policy로 둔다. 제품 메시지는 “단일 기간”이 아니라 “증거 유형별 보존정책”으로 말한다. [[D5a-결과]] [[D15-결과]]

## 제품·심사 매핑
- **1=주제:** 금융권 생성형 AI의 핵심 병목인 망분리·PII 비반출·승인우선 문제를 정면으로 푼다. 2024~2026 정책 흐름과 맞다. [[D5b-결과]] [[D12-결과]]

- **2=금융가치:** 전북은행+JB우리캐피탈 RM이 외부 프런티어 LLM의 추론력을 쓰되, 고객 원본 PII·개인신용정보를 외부로 보내지 않는 내부 콘솔로 설명한다. “업무 생산성”과 “규제 방어”를 같이 잡는다. [[D5a-결과]] [[D10-결과]]

- **3=Agent설계:** Agent는 실행자가 아니라 제안 엔진이다. Skill은 최소권한 툴이고, 고위험 실행은 Approval이 막는다. 툴 브로커·정책 엔진·감사원장이 에이전트 권한을 실제로 제한한다. [[D10-결과]] [[D15-결과]]

- **4=MVP검증:** 데모는 “PII 포함 프롬프트 입력 → 토큰화 → 외부 전송 허용/차단 판정 → 응답 재식별 스캔 → 승인 대기열 → 감사원장”까지 한 흐름으로 검증한다. [[D5a-결과]] [[D10-결과]] [[D15-결과]]

- **5=혁신·운영:** 혁신은 모델이 아니라 운영계약이다. 샌드박스, 금융보안원 클라우드 안전성 평가, ISMS-P, 사고보고, forensic export까지 이어지는 운영 로드맵을 제시한다. [[D12-결과]] [[D15-결과]]

## 시연 한 컷
- RM이 전세대출 상담 메모를 붙여 넣는다. 화면이 이름, 계좌번호, 주민번호를 즉시 감지하고 `[PT_ID_9812]` 같은 토큰으로 바꾼다. 외부 LLM 라우트에는 “가명·토큰화된 최소정보”만 나간다. 응답이 돌아오면 DLP가 준식별자 조합을 다시 스캔한다. 고위험 문구는 승인 대기열에 걸리고, 승인자는 근거 묶음·모델 버전·정책 충돌·반출 스캔 결과를 보고 승인/반려/재심을 선택한다. 아래에는 Audit이 해시와 함께 쌓인다. [[D5a-결과]] [[D10-결과]] [[D15-결과]]

## canon 갱신 후보
- “외부 LLM 비반출” 정의를 “원본 PII·개인신용정보는 외부 모델로 반출하지 않고, 외부에는 가명·토큰화된 최소정보만 위탁 처리 범위 안에서 라우팅하며, 출력 재식별 위험을 스캔하고 감사원장에 남기는 구조”로 올린다. [[D5a-결과]]

- “승인우선” 정의에 **근거 묶음, 이유 코드, 이중승인, 반대·재심·상향보고, SLA, kill switch, immutable evidence**를 포함한다. [[D15-결과]]

- “망분리 대응” 문구를 “망분리 폐지”가 아니라 **망분리 원칙 + 예외 + 보안통제 + 결과책임**으로 고정한다. [[D5b-결과]]

- “검증 로드맵”에 **온프레미스 비식별화 PoC → 혁신금융서비스 공동 신청 → 금융보안원 클라우드 안전성 평가 → ISMS-P 범위 갱신 → 내부 업무망 파일럿**을 후보로 둔다. [[D12-결과]]
