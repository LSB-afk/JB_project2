# 연결-1 — 제품 척추 — model-agnostic 거버넌드 OS + 경제성 증명(NPV·자본구제)

## 연결된 발견
D20은 금융 AI의 주장을 뒤집는다. LLM은 신용평가·FDS·조기경보·문서IE의 “주 모델”이 아니라 특화모델·규칙엔진·그래프·RPA를 호출하는 오케스트레이터다. 그래서 에이전트는 챗봇이 아니라 운영체제여야 한다. [[D20-결과]]

D22는 이 운영체제가 왜 model-agnostic이어야 하는지 설명한다. 한국 금융권의 현실적 해법은 “국산 우선, 로컬 운영 가능한 오픈웨이트, 외부 premium API는 예외적 승격 라우팅”이다. 민감정보 비반출 기본경로와 통제된 승격이 동시에 필요하다. [[D22-결과]]

B1은 그 운영체제의 서버 계약을 구체화한다. `Case → AgentRun → Evidence → Approval → Audit`는 StackStorm의 `paused/pending` 승인 흐름, Temporal의 append-only Event History와 Signal 승인 패턴으로 방어된다. [[B1-결과-gpt]]

D23 두 결과는 “좋은 제품”을 “돈 되는 제품”으로 바꾼다. Gemini는 3년 NPV 33.8억 원, RWA 181억 원 경감, 연 1.9억 원 자기자본비용 절감, 총 리스크 조정 편익 연 5.59억 원을 제시했다. GPT55Pro는 더 보수적으로 3년 NPV 24.2억 원, ROI 471%, 회수기간 5.4개월, IRR 269%를 제시했다. [[D23-결과-gemini]] [[D23-결과-gpt55pro]]

## 1차 인사이트
- LocalGuard OS의 핵심은 “어떤 LLM을 쓰나”가 아니다. 핵심은 모델·룰·스킬·승인·감사를 같은 운영계약에 꽂는 하네스다. 이 구조라야 LLM을 오케스트레이터로 낮추고, 신용·FDS·문서·조기경보는 검증 가능한 전용 엔진에 맡길 수 있다. [[D20-결과]] [[B1-결과-gpt]]

- PII 비반출은 비용 최적화가 아니라 라우팅 정책이다. 순수 비용만 보면 JB형 사용량에서는 API가 유리하지만, 민감 데이터·망분리·데이터주권 요구가 강하면 로컬 1차 처리와 외부 2차 승격이 맞다. 제품 메시지는 “외부 LLM 금지”가 아니라 “외부 호출은 정책적으로 승격된다”여야 한다. [[D22-결과]] [[D23-결과-gpt55pro]]

- 경제성의 본체는 토큰비가 아니다. D23-GPT55Pro는 가장 민감한 변수를 실현율 `R`, 고빈도 업무 건수, 건당 절감시간으로 봤다. D23-Gemini는 M/G/c로 21명 필요 인력을 9명으로 낮추고 연 15.0억 원 절감을 계산했다. 즉 심사위원에게 보여줄 것은 “모델 단가표”가 아니라 “대기열, 승인대기, 재작업, 자본구제”다. [[D23-결과-gpt55pro]] [[D23-결과-gemini]]

- 자본구제는 제품 척추의 킬러 논리다. 단순 업무자동화는 누구나 말한다. 하지만 PD 2.50%→1.80%, LGD 45.0%→42.0% 가정으로 EL 3.69억 원 절감, RWA 181억 원 경감, 자본비용 1.9억 원 절감을 연결하면 은행 경영진 언어가 된다. [[D23-결과-gemini]]

- MVP의 “OS다움”은 UI보다 이벤트 원장에 있다. StackStorm식 승인 큐와 Temporal식 append-only 이벤트를 localStorage mock으로라도 보여주면, 현재 바닐라 MVP가 나중에 서버계약으로 승격될 수 있음을 증명한다. [[B1-결과-gpt]]

## 모순·텐션
- TCO 결론이 갈린다. D23-Gemini는 월 97억 토큰, 약 9.2만 건부터 8x H100 온프레가 API보다 유리하다고 봤다. D23-GPT55Pro는 2×H100 기준 월 약 660만원, 손익분기점을 GPT-5 mini 74.1억 토큰, GPT-5 14.9억 토큰, Sonnet 4 8.17억 토큰으로 봤다. reconcile: “온프레 우위/열위”를 단정하지 말고, GPU 구성·캐시율·모델단가·PII 등급별 라우팅 표로 제시한다. [[D23-결과-gemini]] [[D23-결과-gpt55pro]]

- NPV 규모가 다르다. D23-Gemini는 기준 33.8억 원, D23-GPT55Pro는 기준 24.2억 원이다. reconcile: 발표에는 보수값 24.2억 원을 기본 헤드라인으로 쓰고, 자본구제 포함 확장 시나리오로 33.8억 원을 보조한다. [[D23-결과-gemini]] [[D23-결과-gpt55pro]]

- API 우선과 PII 비반출이 긴장한다. D23-GPT55Pro는 순수 비용 기준 API 우위를 말하고, D22는 민감업무 로컬 1차 처리를 말한다. reconcile: 비용 기준은 비민감·비식별·공개정보 승격에만 적용한다. 고객식별정보·상담기록·내부 규정은 로컬/내부망 경로를 기본값으로 둔다. [[D22-결과]] [[D23-결과-gpt55pro]]

## 제품·심사 매핑
- 1=주제: “지역은행 거버넌드 AI 에이전트 콘솔”을 챗봇이 아니라 금융 AX 운영체제로 정의한다. `Case → AgentRun → Agent → Skill → Evidence → Approval → Audit`가 제품 척추다. [[D20-결과]] [[B1-결과-gpt]]

- 2=금융가치: 인건비 절감, 대기행렬 SLA, EL 절감, RWA 경감, 자기자본비용 절감을 하나의 NPV 모델로 묶는다. 보수 헤드라인은 3년 NPV 24.2억 원, 확장 시나리오는 33.8억 원으로 둔다. [[D23-결과-gpt55pro]] [[D23-결과-gemini]]

- 3=Agent설계: LLM은 주 모델이 아니라 라우터다. `Score Skill`, `Legacy Adapter Skill`, `Tool Registry`, `Policy Engine`, `Audit Skill`을 분리해 모델 교체와 승인정책을 독립시킨다. [[D20-결과]] [[D22-결과]]

- 4=MVP검증: 승인 대기, tool/action preview, evidence attachment, audit event append log, case timeline을 MVP에서 한 흐름으로 보여준다. StackStorm/Temporal식 계약을 바닐라 MVP에 먼저 모사한다. [[B1-결과-gpt]]

- 5=혁신·운영: model-agnostic 구조가 혁신이고, PII 비반출·승인우선·append-only 감사가 운영성이다. 심사위원에게 “모델 성능”보다 “같은 업무를 로컬·국산·오픈웨이트·외부 premium으로 정책 라우팅해도 감사계약은 유지된다”를 보여준다. [[D22-결과]] [[B1-결과-gpt]]

## 시연 한 컷
- RM이 SME 여신 케이스를 연다. LocalGuard OS가 고객 PII를 외부 호출 차단으로 표시하고 로컬 1차 모델로 서류를 요약한다. `Score Skill`이 PD/LGD 변화와 RWA 영향, `Evidence`가 근거 문서, `Approval`이 승인 대기 액션, `Audit`가 append-only 이벤트를 한 화면에 쌓는다. 우측 경제성 패널에는 “RWA 181억 원 경감 / 연 1.9억 원 자본비용 절감 / 3년 NPV 24.2억~33.8억 원”을 시나리오로 보여준다. [[D23-결과-gemini]] [[D23-결과-gpt55pro]] [[B1-결과-gpt]]

## canon 갱신 후보
- “에이전트는 금융업무의 주 모델이 아니라 특화모델·규칙엔진·검색·RPA를 호출하는 운영체제” 문장을 제품 정의에 추가한다. [[D20-결과]]

- “model-agnostic = 로컬 1차 + 외부 2차 승격 + 동일 감사계약 유지”를 아키텍처 원칙으로 추가한다. [[D22-결과]]

- 경제성 기준값은 보수 헤드라인 `3년 NPV 24.2억 원, ROI 471%, 회수기간 5.4개월`로 올리고, 자본구제 확장 시나리오 `3년 NPV 33.8억 원, RWA 181억 원 경감, 연 1.9억 원 자기자본비용 절감`을 보조 수치로 둔다. [[D23-결과-gpt55pro]] [[D23-결과-gemini]]

- OSS 롤모델은 paperclip을 1차 본보기로 두되, 서버계약 보조 근거로 StackStorm `Approval`, Temporal `Audit/Event History`, Carbon/Tremor `콘솔 규율`을 명시한다. [[B1-결과-gpt]]
