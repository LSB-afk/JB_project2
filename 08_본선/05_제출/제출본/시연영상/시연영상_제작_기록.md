## 장면 구성

1. 와이드 RM 포털 진입: 좌측 역할 메뉴, 중앙 업무보드, 우측 상세 패널이 한 화면에 보이도록 1920x1080/90% 구도로 촬영
2. 업무보드 우선순위: 처리 전·중·후, 위험 신호, SLA, 근거 데이터를 동시에 확인
3. AI 인풋 접수: 비정형 상담 메모가 케이스, 트리거, 서브케이스, 실행 준비 단계로 분해되는 과정 확인
4. 케이스 상세: 생성된 케이스를 열어 위험 신호, 다음 액션, 에이전트 배정, 산출물 탭이 우측 상세 패널과 함께 바뀌는 장면 확인
5. AI 실행 큐: 케이스별 에이전트가 승인 대기·실행 중·완료 상태로 병렬 처리되는 구조 확인
6. 에이전트 하네스: 오케스트레이터, 도메인 Agent, 가드레일, 사람 검토, 감사 기록의 실행 흐름 확인
7. 승인 라우팅: 고객 영향이 있는 액션은 승인권자 검토를 통과해야 실행되는 구조 확인
8. 통합 리포트: 개별 에이전트 산출물과 통합 MD, 직원 확인 포인트 확인
9. 감사 기록: 접수, 분해, 실행, 승인, 산출물 생성 이벤트가 감사 로그에 남는 장면 확인
10. 요약 카드: AI는 판단 후보와 초안을 만들고, 사람은 승인과 책임을 남기는 원칙 정리

## 제작/검증 결과

- 영상 길이: 4분 00초
- 해상도: 1920x1080
- 코덱/컨테이너: H.264 MP4
- 파일 크기: 약 29MB
- 제출 제한 확인: 5분 이하, 100MB 이하
- 검증:
  - `node --check scripts/record_finals_demo_remote.mjs`
  - `npm run build`
  - `E2E_PORT=8128 npm run test:e2e -- tests/e2e/role-consoles.spec.js --project=desktop`
  - MP4 전체 디코딩 검사
  - 7200프레임/30fps/240초 확인
  - 주요 시점 프레임 육안 확인: 업무보드, 인풋 분해, 케이스 상세, 실행 큐, 통합 리포트, 종료 요약 카드

## 확인 사항

- GitHub Pages 최신 배포 화면을 기준으로 다시 녹화했다.
- 화면 안에 더 많은 레이아웃이 들어오도록 1920x1080 캔버스와 90% 녹화 전용 스케일을 적용했다.
- 원격 배포 환경에서는 로컬 Agent API(`127.0.0.1`) 호출이 CORS로 차단될 수 있어, 영상은 UI 플로우와 승인/보고/감사 흐름 중심으로 구성했다.
