## 2. 보너스 섹션 — 킬러컷 3종 (4단계 뒤, Q&A·여유 시간 대응용)

> 덱 4단계 골든패스가 끝난 뒤 붙이는 별도 파트. 화면이 CCL 콘솔에서 터미널로 전환되므로 **본편과 물리적으로 분리된 테이크**로 찍는다(§0 Docker/게이트웨이 전제 참고).

> **K2는 [구현 대기 — 촬영 전 코드 작업 필요].** `scripts/llm-gateway.mjs:28`의 `ladderFor(first) = [...new Set([first, "claude", "codex"])]`는 `first="codex"`일 때 사다리가 `[codex, claude]`로만 뻗는다 — **ollama가 안 들어있어서, 지금 코드로는 codex 요청이 실패해도 자동으로 로컬모델(ollama)까지 안 넘어간다.** "코덱스→로컬모델 전환"을 한 번의 `/llm` 호출 안에서 `fallbackPath`로 실제로 찍으려면 이 사다리에 ollama를 추가하는 코드 변경이 먼저다(예: `first="codex"`일 때 `[codex, ollama, claude]`로). 코드 수정 전까지는 아래 K2를 **수동 2단계 curl**(engine을 바꿔가며 각각 호출)로 대체 촬영할 수 있으나, 그러면 같은 `runId`의 `fallbackPath`가 아니라 서로 다른 두 런의 `engine` 필드 비교가 된다 — 자막에서 "자동 전환"이라 단정하지 말고 팀 구현 완료 여부에 맞춰 문구 확정.

### 3-1. `record_demo.mjs` 재사용 판정 — **재사용 불가, 골격만 복제**

`02_제품/scripts/record_demo.mjs`는 `BASE = http://127.0.0.1:8011`(예선 앱 전용 포트), `[data-view]` 속성, `#cd-run` 버튼, `.gov-panel`/`.cd-timeline` 클래스, `window.openCaseDetail()` 전역 함수, `?demo=sme` 쿼리 등 **02_제품/app(예선 MVP)에만 존재하는 DOM/전역**을 직접 참조한다. JB_project2의 CCL 콘솔은 DOM 구조·해시 라우팅·전역 함수가 전부 다르므로(`[data-ccl-open-case]`, `[data-role-filter]`, `#/roles/corporate-credit/...`) **이 스크립트를 그대로 못 쓴다.**

재사용 가능한 것은 **패턴**뿐이다: Playwright `chromium.launch()` + `context.newContext({ recordVideo })` + `page.evaluate()`로 body에 자막 오버레이(`#demo-cap`)와 타이틀 카드(`#demo-card`)를 주입하는 골격, 그리고 `cap()`/`card()`/`go()` 헬퍼 함수 형태. → **신규 파일 `record_demo_ccl.mjs`를 이 골격을 복사해 새로 작성**(BASE `127.0.0.1:8000`, 셀렉터를 위 §1 표의 실제 값으로 교체)하는 것을 권장한다.

```bash
  -d '{"prompt":"한 문장으로: 전세가율이란?","engine":"codex","caseId":"CCL-0001"}' | jq .
# → codex를 강제로 죽인 상태라면 응답에 fallbackPath:["codex","ollama"]가 찍힘

# ② 로그 확인 (양쪽 방식 공통)
tail -f scripts/llm-runs.jsonl | jq .

# K3 — 토큰 계기판
curl -s localhost:8022/llm/usage | jq .
```

K2 리스크: 촬영 시점에 `ladderFor` 코드 수정이 아직 안 됐다면 ①-A(수동 2단계)로 찍고 자막에서 "자동 전환"이라 단정하지 않는다. 로컬 Ollama에 모델이 안 받아져 있으면 K2 자체가 실패하니 리허설에서 `ollama list`로 모델 존재를 먼저 확인할 것.

### 3-4. 자동 vs 수동 체크리스트

| 씬 | 자동(Claude/Playwright) | 수동(사람) |
|---|---|---|
| 0(타이틀) | ✅ | — |
| 1~9(CCL 골든패스) | ✅ `record_demo_ccl.mjs` | 셀렉터·approval id 리허설 1회 확인 필요 |
| K1(Docker) | — | ✅ Docker Desktop 기동·화면 녹화 |
| K2(코덱스→로컬모델) | — | ⛔ **선행 조건**: `scripts/llm-gateway.mjs`의 `ladderFor`에 ollama 경로 추가(팀 구현) → 이후 ✅ 터미널 화면 녹화, ollama 모델 로딩 상태 사전 확인 |
| K3(토큰) | — | ✅ 터미널 화면 녹화 (K2와 같은 테이크로 이어 찍어도 됨) |
| C(클로징) | ✅ | — |
| 자막 합성·컷 편집·mp4 최종 인코딩 | ✅ (ffmpeg 스크립트화 가능) | 최종 육안 검수 1회 권장 |

자동 캡처 9개 유닛(0~9, C) vs 수동 3개 유닛(K1~K3) — **12개 중 9개 자동, 3개 수동**.

## 4. 리스크 · 폴백

- **approval id 불일치**: `cclConsole.data.js`의 시드 approval id(`APR-CCL-0001/0002`)가 리허설 중 다르게 재생성될 수 있음 — `localStorage` 초기화 직후 첫 로드 상태로만 촬영(중간 상태 저장 금지).
- **K2 결과 불확실성**: 클로드/코덱스 CLI 인증 상태에 따라 성공/전체실패 두 결과 중 하나가 나온다 — 스토리보드는 성공(claude로 폴백) 기준이나, 실패해도 "사람 큐 격상" 증거로 캡션만 바꿔 쓸 수 있음.
- **Docker가 심사 환경에서 안 뜨는 경우**: 런북(`시연-런북-백엔드분리.md` §실패 대비)대로 사전 녹화본 정적 컷으로 대체.
- **5분 초과 방지**: K1~K3 중 시간이 빠듯하면 K3(토큰 계기판, 13초)을 가장 먼저 자막 카드 1장(스틸컷)으로 축소.
