# 표지·클로징 카피 확정 — JByond 태그라인 배치

> 2026-07-05 총괄 판단. [[발표자료-피그마-정합성체크]] §1 프레임 판독(실제 덱 14프레임) 기준으로 배치 위치를 정함.

## 결정

"JByond — 다음 세대를 잇는 JB금융의 AX Operating System" 태그라인은 **표지(프레임1, node 4915-2719)**에 넣는다. 클로징(프레임13, node 4977-5378)에는 새로 넣지 않는다.

## 근거

- 표지는 이후 모든 슬라이드(문제→솔루션→기능→효과)를 심사위원이 읽는 틀을 만든다. 포지셔닝 문구를 먼저 주면 뒤 내용이 그 틀 안에서 해석된다. 클로징에서 처음 던지면 이미 각자 정리한 인상 위에 얹는 꼴이라 임팩트가 약함.
- 실제 표지(프레임1) 카피는 "JByond" 타이틀 + 기능 설명형 부제("AI 에이전트가 근거를 정리하고, 사람은 승인하며...")뿐 — 정체성/포지셔닝 문구가 비어 있어 태그라인이 들어갈 자리가 이미 있음.
- 실제 클로징(프레임13) 카피는 이미 "JB금융이 AX 전환 기반을 갖춘 상태로 나아가게"로 미래지향 메시지를 담고 있음 — 별도 태그라인을 또 넣으면 마무리 멘트가 두 갈래로 갈림.
