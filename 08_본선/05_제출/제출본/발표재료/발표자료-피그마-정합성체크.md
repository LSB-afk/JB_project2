## 2. 핵심 결정사항 (덱에서 확정된 것 — 다른 산출물이 따라야 함)

1. **제품명 = JByond** (구 JB LocalGuard OS — 표지·확장 슬라이드 모두 JByond). 기능명세서·스토리보드·벼락치기 모두 "JByond(구 JB LocalGuard OS)" 표기로 전파 완료.
2. **팀 표기 = Go Lab** / 팀원 이승보·김주용·김민주·이재형. ⚠️ 접수 팀명(망상궤도 vs Go Lab)은 사람 확인 필요.
3. **시연 4단계 골격** 고정: 케이스 확인 → Agent 실행 → 사람 검토(Enter) → Audit Log.
4. **덱 공식 수치**: 360건/20분/55%/66시간 · NPV 24.2억/ROI 471%/5.4개월 · 28.1분→3.7분(87%) · 에이전트 14종/스킬 25종 · L0~L4.
5. 히어로 케이스 = **CCL-0001 전주 중앙로 카페 운전자금** (User Scenario 슬라이드와 코드 데모 경로 일치 ✅).

## 3. 정합성 플래그 (사람/후속 확인)

- 🚨 **슬라이드 10 "고객 대기 28.1분→3.7분(87%)" 수정 권고** — 팀 자체 검증(`문제정의-기대효과-보강.md` §5·§6-1)에서 JB 데이터가 아닌 **큐잉이론 가상 예시 파라미터**로 판명, 헤드라인 사용 비권장 결정된 수치. 덱에서 빼거나 "큐잉이론 시뮬레이션 예시"로 문구를 바꿔야 함. 대체 카드: 시스템 실측 응답 1.0~1.1초(Playwright). 기능명세서·벼락치기에는 이미 정정 반영 완료.

- ⚠️ **"Enter Interaction 승인"(슬라이드 4·12) vs 코드 현실** — Enter 키 승인은 RM 콘솔(rmoHandleKeydown)에만 구현, 데모 히어로 CCL-0001이 속한 CCL 콘솔은 마우스 클릭 승인뿐(keydown 핸들러 없음). 시연에서 Enter 승인을 화면으로 못 보여줌 → 팀 결정 필요: (a) 시연 일부를 RM 콘솔로 전환해 Enter 승인 1컷 확보, 또는 (b) 시연 자막·발표 멘트에서 "Enter 승인"을 "원클릭 승인"으로 완화. 스토리보드는 (b) 기준으로 정직하게 작성됨.
- ⚠️ **에이전트 14종·스킬 25종**(덱) vs 코드 실측 58/40([[구현현황-JB_project2]]) — 덱은 "조직도에 보여주는 14종" 관점. Q&A에서 "코드에는 58종 등록, 조직도에는 14종 노출" 방어 준비(벼락치기 시트에 반영 지시함).
- ⚠️ 슬라이드 7 모델 레이어에 "[제휴 TBD]" 잔존 — 발표 전 문구 확정 필요.
- ⚠️ 팀명 표기 Go Lab ↔ 접수 팀명 확인.
- 슬라이드 3 기사 인용(2026-06-26 사업자대출 사후점검 강화)은 발표 멘트에서 출처(쿠키뉴스·은행연합회) 언급 가능하게 벼락치기에 포함.
