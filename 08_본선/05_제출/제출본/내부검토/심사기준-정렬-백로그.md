## 3. 배포 레포 구조안

### 마스터 폴더 이름 후보 3개
1. **`localguard/`** — 기존 확정 제품명 "JB LocalGuard OS"와 즉시 일치, 브랜드 혼선 없음, canon 전체와 정합. **추천.**
2. `caseops/` — 운영계약 `Case→AgentRun→...→Audit`을 직접 반영(내부적으로 "CaseOps 분기" 문서군 이미 존재), 다만 대외 발표 브랜드명과는 별개 용어라 설명이 한 단계 더 필요.
3. `guardline/` — "승인 게이트 라인" 이미지, 신규 조어라 설명 비용 발생.

paperclip 롤모델 확인 결과: paperclip은 레포 루트 자체가 이미 하나의 제품 단위이며 `docs/ evals/ packages/ server/ skills/ ui/ scripts/` 등을 **최상위에 나란히** 두고, `README.md·AGENTS.md·ROADMAP.md·CONTRIBUTING.md`로 기획력·거버넌스를 루트에서 바로 보여준다. 같은 패턴을 `localguard/` 하나로 감싸 적용한다.

### README 최상단 개요 — 무엇을 보여줄지
1. **한 줄 문제정의 + 히어로 케이스**(CCL-0001, 전주 카페 여신) — 기획력.
2. **운영계약 다이어그램**(`Case→AgentRun→Agent→Skill→Evidence→Approval→Audit`) — 설계.
3. **자가채점 요약**(상14/중8/하3, `reports/`로 링크) — 과장 없는 정직성 자체가 심사 인상 포인트.
4. **비전 한 줄**("AI는 사람이 사람답게 판단할 여유를 만든다") + 원칙 링크.
5. **협업 증거**: fork(River-181)→PR#1/#2, issue #3 스크린샷/링크 — 실제 다인 GitHub 협업 흐름을 숨기지 않고 노출.
6. **Quick start**: `npm install / npm run dev / npm run test / npm run test:e2e`.
