# Compliance Rules — 규제·법률 준수 규칙

> 신뢰마커: **[확정]** = `_canon.md` §4 / [[08_본선/03_제품/01_결정-준비/근거팩/적법성-근거팩|적법성-근거팩]] 검증 완료. **[리서치 검증]** = 근거팩이 "canon [확정]은 아니나 리서치(D14/D15 등)로 검증"이라 명시. **[미검증]** = 근거팩 자체가 "제출 직전 재확인 필요"로 표기한 항목 — **이 문서에서 그대로 승계**하며 새로 만들지 않는다. **E4** = 코드 가드레일 존재.
> 법조문은 근거팩 원문 그대로 인용한다(패러프레이즈 금지) — SSOT: `_canon.md` §4, [[08_본선/03_제품/01_결정-준비/근거팩/적법성-근거팩|적법성-근거팩]], [[08_본선/03_제품/01_결정-준비/근거팩/국내-AI규제-HITL-딥리서치|국내-AI규제-HITL-딥리서치]].

|---|---|---|---|

| CMP-02 | 신용정보법 §40조의2 ①②⑧ | 가명처리 + 토큰↔원본 키 분리보관·안전조치·처리기록 | [확정] |
| CMP-03 | 신용정보법 §40조의2 ⑥⑦ | 재식별 금지, 재식별 시 즉시 중단·회수·파기 | [확정] |
| CMP-04 | PIPA §28조의4·§28조의5 | CMP-02·CMP-03의 보충 근거 | [확정] |
| CMP-05 | 전자금융감독규정 §15조 ①항(3·5호) | 망분리 예외 요건(외부/클라우드 LLM 활용 정당화) | **[미검증]** — 정확한 호수는 제출 직전 law.go.kr 재확인 필요(근거팩 승계) |
| CMP-06 | 신용정보법 §36조의2 | 자동화평가(AI 판단) 설명요구권 — §40조의2와 **별개 조문** | [리서치 검증](D14/D15) |
| CMP-07 | 「인공지능 발전과 신뢰 기반 조성 등에 관한 기본법」 §2조 제4호 사목 | 대출 심사 등은 "고영향 인공지능"에 해당 | [리서치 검증](국내-AI규제-HITL-딥리서치 §2.4) |
| CMP-08 | 동법 §34조 ①항 | 고영향 AI 사업자는 사람의 관리·감독(4호) + 조치 문서 작성·보관(5호) 의무 | [리서치 검증] |
| CMP-09 | 금융위 2026「금융분야 인공지능 가이드라인」 보조수단성 원칙 | "인공지능은 업무의 보조수단이므로 최종 의사결정과 그에 따른 책임은 임직원이 수행" | [리서치 검증], 원문 인용(p.29) |
| CMP-10 | 동 가이드라인 RACI 준수사례(p.30) | "'인공지능 모델 점수 산출'은 '권고', '산출 점수 및 증빙서류 검토'(여신심사역)는 '실무 책임', '최종 승인'(지점장/심사센터장)은 '최종 책임'" | [리서치 검증], 원문 인용 |

> CMP-06과 CMP-02·CMP-03는 별개 조문이며 하나가 다른 하나의 오표기가 아니다 — [[08_본선/03_제품/01_결정-준비/근거팩/적법성-근거팩|적법성-근거팩]] 서두가 명시적으로 정정한 오탐 이슈.

## 8. 미검증 표기 승계 (Verify-Before-Submit)

- CMP-05(전자금융감독규정 §15조 ①항 3·5호): 정확한 호수는 제출 직전 law.go.kr 재확인 필요 — [[08_본선/03_제품/01_결정-준비/근거팩/적법성-근거팩|적법성-근거팩]] 원문 [미검증] 마커 그대로 승계.
- CMP-23·CMP-24(보관기간·연합학습 수치): 확정값 아님, 발표·제출물에 확정처럼 쓰지 않는다.
- 확인 필요(사용자 검증 권장): 이 문서가 인용한 금융위 2026 가이드라인 페이지 인용(p.29~34)은 [[08_본선/03_제품/01_결정-준비/근거팩/국내-AI규제-HITL-딥리서치|국내-AI규제-HITL-딥리서치]]가 "feynman이 실제 FSC PDF를 로컬 파싱해 인용, 핵심 인용 4곳(RACI/보조수단성/Override사유입력/Kill switch)은 직접 검증"이라 밝히나 "EU/US 최신개정·세부 페이지 인용은 발표 전 팀 원문 재대조 권장"이라 표기 — 발표 전 재대조 대상.
