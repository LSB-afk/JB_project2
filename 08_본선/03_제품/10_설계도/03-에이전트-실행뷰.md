# 03. 에이전트 실행뷰 (S-05 / S-03-inline)

> SSOT: `06-정보체계-뷰-데이터바인딩-스펙.md` §3.5(에이전트실행뷰) · §3.16(에이전트 실행 애니메이션·비동기 델타) · `2026-07-04-전략회의2-메인대시보드-정리본.md` 결정 4·6·8·9 · `기능요구사항-7-4회의.md` FR-18·19·20·21·22.

## 1. 목적 · 진입

- **목적**: AI 에이전트가 "회사 직원처럼" 무엇을 입력받아 어떤 스킬로 실행하고 어디로 handoff했는지 보여주고, RM이 **엔터만으로** 하위 에이전트들을 순서대로 검토·승인하며 밀어낼 수 있게 한다.
- **진입**: [[02-케이스-상세]] `runs` 탭 안(인라인, `S-03-inline`) 또는 독립 화면(`S-05`, 케이스보드에서 실행 이력만 볼 때). 이 문서는 두 진입 모두에 적용되는 실행 카드 자체의 상세 스펙이다.
- **역할 분기**: `reviewer`는 실행 요청·엔터 승인 가능. `supervisor`/준법은 결과 검토와 차단 사유 확인만(승인 자체는 `S-04-detail`). 관리자만 agent config 변경.

**엔터 승인 애니메이션 시퀀스** (FR-18·20):

```text
1. 포커스 카드에서 Enter
2. 카드 테두리 스트로크(강조) — "승인됨" 시각 확인
3. 카드가 딤드(흐림) 처리
4. 카드가 아래로 슬라이드(완료 큐로 이동, 다크 처리)
5. 다음 대기 카드가 위로 슬라이드해 포커스 위치 획득 + 붙는 이유/기대값 펼쳐짐
```

## 5. 상태

| 상태 | 표현 |
|---|---|
| Empty | "실행된 에이전트 없음" |
| Loading(queued/running) | 스피너만(퍼센트 없음), `BackgroundNavHint` 노출 |
| Error | LLM 실패 또는 guardrail 위반 배너 |
| Fallback | live output 실패 시 모의 결과 유지 + `needsReview` badge |
| 완료(다크) | 승인 완료된 카드는 큐 하단으로 이동, 어두운 색조 |
| 알림 도착 | `NotificationChime`(띠링) — 다른 케이스를 보던 중에도 상단 고정 알림으로 노출 |

## 6. 키보드 · 인터랙션 맵

| 키 | 동작 | 2단 확인 | 전환 |
|---|---|---|---|
| `Enter`(포커스 카드) | 실행/승인 — "통과, 통과" | **일반**: 단일 Enter. **고위험 산출물**: `Tab`으로 확인 버튼까지 이동 후 명시 활성화(§2.3.1 a11y, 엔터 단독 금지) | 스트로크→딤드→아래로, 다음이 위로 |
| `↓`/`Space` | 다음 대기 카드로 포커스만 이동(실행 안 함) | 없음 | 없음 |
| 숫자키(예: `2`) | 큐 N번째 에이전트 카드 직접 열기 | 없음 | 포커스 이동 |
| `Space`(포커스 카드) | 붙는 이유/기대값/transcript 펼치기 토글 | 없음 | 없음 |
| `Shift+Enter` | 반려/재검토 요청(사유 필요) `[분기/미확정]` | 사유 없으면 제출 차단 | 카드 재순환(대기 큐로 복귀) |
| `Esc` | `BackgroundNavHint`로 다른 케이스 이동(실행은 백그라운드 유지) | 없음 | 페이지 전환, 실행 상태 유지 |
| `Tab` | 접근성 순차 포커스, 고위험 승인의 필수 확인 경로 | — | — |

- **"엔터에 과한 책임 금지"**(김민주 05:02): 일반 하위 에이전트 실행/승인은 단일 Enter로 진행하되, 산출물이 `riskLevel=high/critical`이면 §2.3 예외(마우스 확인 또는 2단 확인 강제)를 그대로 적용한다 — Enter 한 번으로 고위험 결정까지 끝나지 않는다.
- **비동기 처리**: Enter 직후 로딩은 화면을 막지 않는다. RM은 `Esc`/`BackgroundNavHint`로 다른 케이스를 보다가, 완료되면 `NotificationChime`으로 돌아온다(예: 3번 케이스 보던 중 1번 완료 알림 → 1번으로 돌아가 승인 → 2번).
