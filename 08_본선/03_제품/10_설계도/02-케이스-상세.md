# 02. 케이스 상세 (S-03-detail)

> SSOT: `06-정보체계-뷰-데이터바인딩-스펙.md` §3.4(케이스상세) · §3.16(결과물 MD 뷰어 스키마) · `기능요구사항-7-4회의.md` FR-07·08·18·19·23·24.
> 히어로 데이터: `CCL-0001` — 전주 카페 운영자 운전자금 검토.

## 1. 목적 · 진입

- **목적**: 한 `Case`의 상태·근거·실행·승인·감사가 끊기지 않고 한 화면에서 이어지게 한다(FR-07 "다중 에이전트·다중 스킬·외부데이터 통합 뷰"). **위=케이스 정보, 아래=에이전트·API 정보**로 계층을 나눠 헷갈리지 않게 한다(FR-18).
- **진입**: [[01-메인-업무보드]]에서 `CaseCard` Enter/클릭 → 이 화면. 또는 `S-04` 승인대기함의 `ApprovalCard`에서 연결된 Case로 진입. 또는 `S-13` 감사 행의 `ccl_case` 타깃 링크.
- **역할 분기**:
  - `reviewer`: 기본 진입 탭 = `signals`(위험 신호). 실행 요청/초안 작성 가능.
  - `준법(compliance)`: 기본 진입 탭 = `evidence`(근거, PII 검증 우선).
  - `supervisor`: `runs`(실행) 우선 확인 후 승인/반려 CTA(`ApprovalStatusStrip`) 사용.

## 4. 데이터 바인딩

| 영역 | 테이블/필드 · ViewModel | R/W | 근거(§) |
|---|---|---|---|
| `CaseDetailHeader`/`InfoGrid` | `CaseDetailBundle` = `Case + Signal[] + EvidencePack + AgentRun[] + ApprovalRecord[] + AuditEvent[]` | R | §1.4 |
| `signals` 탭 | `Signal[]`, `RiskDecision`(목표 모델), `ccl_cases.riskLevel/repaymentBand/docsStatus` | R/D | §1.2, §3.4 |
| `evidence` 탭 | `ccl_review_notes`, `ccl_doc_checks`, `ccl_consult_logs`, `ai_recommendations` | R | §1.2 |

| `ResultMdViewer` | 취합 MD(목표, 개별 AgentRun outputSummary 취합) `[분기/미확정]` | R | §3.16 |

## 5. 상태

| 상태 | 표현 |
|---|---|
| Empty | 케이스 없음 또는 scope 밖("이 케이스에 접근 권한이 없습니다") |
| Loading | `CaseDetailBundle` 로드 중 skeleton(탭 전체) |
| Error | "상세 데이터를 찾을 수 없습니다" |
| Stale | 데이터 기준 시각 표시(헤더 우측) |
| 완료(다크) | `doneHold` 상태 케이스는 헤더/탭 전체가 어두운 색조로 렌더(전역 완료=다크 원칙) |
| Fallback | live LLM 실패 시 모의 결과 유지 + `needsReview` badge(`runs` 탭) |
| 폴백(결과물) | 옵시디언식 링크 미구현 시 통합본+개별 MD **분리 표시**로 폴백(§3.16, 김민주 21:38) `[분기/미확정]` |

## 6. 키보드 · 인터랙션 맵

| 키 | 동작 | 2단 확인 | 전환 |
|---|---|---|---|
| `Tab` 또는 `Ctrl+[`/`Ctrl+]` | 탭 순환: `signals → evidence → runs → verify → audit` | 없음 | 탭 콘텐츠 교체 |
| `Space`(포커스 항목) | 근거/미리보기 펼치기 | 없음 | 없음 |
| `Enter`(`runs` 탭, 에이전트 카드 포커스) | 해당 하위 에이전트 실행 요청/승인 — 상세 동작은 [[03-에이전트-실행뷰]] | 없음(단일 실행), 고위험 산출물은 §2.3 예외 | 스트로크→딤드→다음 |
| `Enter`(`MdLinkChip`) | `[[개별MD]]` 열람 → `FloatingMdModal` | 없음 | 모달 오픈, 배경 딤드 |
| `Enter`(`AgentMentionLink`, `@에이전트`) | `S-07-detail` 에이전트 워크스페이스로 이동 | 없음 | 페이지 전환 |
| `Enter`(`ApprovalStatusStrip`) | `S-04-detail`로 이동(이 화면에서 승인/거부 결정 자체는 하지 않음) | 없음 | 페이지 전환 |
| `Esc` | `FloatingMdModal` 닫기 | 없음 | 모달 닫힘, 배경 딤드 해제 |

- **탭 vs 스택 확정 근거**: 세로 스택이면 high/critical 케이스에서 근거·감사가 하단에 묻혀 FR-07 "통합 뷰" 목적이 깨짐 → 탭으로 확정(§3.4, Paperclip `PageTabBar` 패턴 번역).
- **badge dot 기준은 미확정**(`[미확정]`) — "안 본 근거" 판정 로직은 팀 확정 대기(§6 미확정 목록).

## 7. FR 근거

- FR-07 (케이스 상세 = 다중 에이전트·스킬·외부데이터 통합 뷰)
- FR-08 (케이스=마크다운 파일 개념 — `ResultMdViewer`가 부분 구현, 케이스 자체가 파일은 아님 `[미반영/부분]`)
- FR-18 (케이스 드릴인 2계층: 위=케이스 정보/아래=에이전트 정보)
- FR-19 (하위 에이전트 각자 MD → 취합 MD 1개)
- FR-23 (결과물 MD 뷰: 통합본+개별, 플로팅 모달, 스킬·출처 하단)
- FR-24 (옵시디언/노션식 링크, `@에이전트`→에이전트 화면)
- §3.4 탭 확정 근거, §3.16 결과물 MD 뷰어 스키마

**가변 표현**: `InfoGrid` 위치(가로 그리드 vs 우측 패널), 탭 배지에 숫자를 넣을지 dot만 쓸지는 개발 중 바뀔 수 있다.
