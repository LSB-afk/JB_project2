# 01. 메인 업무보드 (S-03 케이스보드) — 플래그십 화면

> SSOT: `06-정보체계-뷰-데이터바인딩-스펙.md` §3.2(케이스보드) · §3.16(전략회의2 델타) · `2026-07-04-전략회의2-메인대시보드-정리본.md` · `기능요구사항-7-4회의.md` FR-05·06·16·17·20·25·27.
> 이 화면은 팀이 손그림으로 확정한 **피그마 예시 화면**(2026-07-04 6:03 회의)의 재현 대상이다. 로그인(`S-00`) 이후 기본 랜딩이며, 조직도(`S-07`)는 메인이 아니다(§6 미확정 해소).

## 1. 목적 · 진입

- **목적**: RM(기업여신 담당자)이 콘솔에 들어오는 순간 "오늘 무엇을, 몇 건, 어떤 순서로 봐야 하는가"를 1초 안에 파악하게 한다. 총 건수가 아니라 **진행 단계별 카운트**(전/중/후)로 보여준다(FR-17).
- **진입 경로**: `S-00` 로그인/역할 선택(계열사=전북은행 또는 JB우리캐피탈, 역할=기업여신 담당자) → **큰 진입 버튼**(뷰 버튼 2개 정도 들어가는 크기, 김민주 10:07~10:24 근거)을 눌러 업무보드로 진입. 이 버튼은 마우스/터치 둘 다로 크게 눌리도록 설계하며, 이후 화면 안에서는 Enter-first로 전환된다.
- **역할 분기**:
  - `reviewer`(기업여신 담당자): 본인/팀 담당 케이스 우선 정렬, 케이스 생성(`S-03-create`) 가능.
  - `supervisor`: `requiresHumanReview=true` 및 `riskLevel=high/critical` 케이스를 상단 고정.
  - `준법(compliance)`: L3/L4 승인 필요 케이스 우선 노출.
  - 대기 중인 승인이 있으면 `S-04` 승인대기함 badge를 강조해 다음 이동을 유도한다(§3.1 다음 이동 규칙).
- **다음 이동**: 케이스 카드 Enter/클릭 → `S-03-detail`(케이스 상세, [[02-케이스-상세]]). 단계 카운트 칩 Enter/클릭 → 해당 단계로 필터된 보드.

- 컬럼 헤더의 카운트는 `[B]`의 전/중/후 합계로 다시 접힌다. 컬럼 자체는 §3.2 컬럼 정합표(6컬럼)를 그대로 쓴다.
- `CaseCard`는 예쁜 요약 카드가 아니라 "오늘 처리 순서가 보이는 업무 행"이어야 한다(§5.1 디자인 지시). 카드 안 모든 badge(승인대기·evidence debt)는 클릭 시 해당 상세 탭으로 바로 이동한다.
- `완료·보류` 컬럼 카드는 **어두운 색**(dimmed/dark)으로 렌더(FR-22 "완료 케이스=어두운 색" 원칙을 보드 레벨에도 적용).

## 6. 키보드 · 인터랙션 맵

| 키 | 동작 | 2단 확인 | 전환 |
|---|---|---|---|
| `↓` / `Space` | 다음 케이스로 포커스 이동(큐 순서, priorityScore순) | 없음 | 포커스 이동만, 화면 전환 없음 |
| `Space`(포커스 상태에서 1회) | 해당 케이스 미리보기(context-panel) 열기 | 없음 | 근거 없음이면 자동 보류 표시 |
| `Space`(2회 연속) | 검토완료(reviewed) 표시 후 다음으로 이동 `[분기/미확정]` | high/critical은 reviewed만, 승인 아님 | 카드 흐림 처리 |
| `Enter` (케이스 카드 포커스) | `S-03-detail`로 진입(대형 진입 버튼의 키보드 등가) | 없음 | 페이지 전환 |
| 숫자키(예: `2`) | 큐 N번째 케이스 직접 열기 | 없음 | 페이지 전환 |
| `StageCountChip`에서 `Enter` | 해당 단계(전/중/후)로 보드 필터 | 없음 | 컬럼 필터 적용 |
| `Ctrl+1`~`Ctrl+5`(또는 대안 `Alt+숫자`/`g 숫자` `[분기/미확정]`) | 뷰 전환: `1`=본 화면, `2`=승인대기함, `3`=에이전트/조직도, `4`=활동/감사, `5`=설정 | 없음 | 전체 화면 전환, Enter-first(케이스 큐 진행)와 별개 축 |
| `Tab` | 접근성 기본 순차 포커스, 모든 Enter-first 동선을 대체 완주 | 위험 결정 전 `Tab`으로 확인 버튼까지 포커스 이동 후 명시 활성화 | a11y 필수, 단순화 금지 |

- **마우스 절대 금지 원칙**은 이 화면부터 적용된다(FR-20). 단, 접근성 기본값인 `Tab`/포인터 입력은 병행 지원한다.
- **엔터에 과한 책임 금지**: 케이스 카드 Enter는 "상세 진입"만 수행하며 승인/거부 등 결정 행위를 하지 않는다. 결정은 `S-04-detail`([[04-승인-대기·결정]])에서만 일어난다.
