# 쇼케이스 빌드플랜 — 풀 쇼케이스(9흐름 × 5페르소나) 도장깨기

> **[대외비]** 본선 시연·영상·발표용 내부 빌드 계획. 인물·거래는 전부 합성(PII無). 외부 공개 전 사람 승인.
> **목적**: 승보 프로젝트2 뷰를 최소 기준으로 두고, **실제 작동하는 흐름 몇 가지**를 선정해 시나리오·시연영상·발표로 1등을 노린다. **최대 풀 쇼케이스**로 잡고 **하나씩 도장깨기**(가능하면 전부).
> **최신 히어로 정정**: 구 히어로 `JBG-104 전주 중앙로 카페(김건우)`는 **폐기**. 최신 히어로 = Figma `Problem Definition`(node 4815-9181)의 **5페르소나 쇼케이스**. 이 계획이 그 반영 SSOT다.

### Figma ↔ 캐논 정합 미결 (팀 확정 필요)
- **계열사 재배치**: Figma 카드는 송민재·윤복례를 **전북은행**으로 표기(캐논=광주은행). 2계열사(전북은행+JB우리캐피탈) 데모 스코프에 맞춘 재배치로 추정 → 캐논 정정할지, 광주은행 유지할지 확정 필요. 문서희(P2)는 Figma도 광주은행 유지(3번째 계열사 카메오).
- **나이 표기**: 송민재 Figma 50 vs 캐논 52 — 경미, 캐논(52) 우선.
- **전세 도메인 부재**: 이전 라이브 슬라이스였던 전세(`?live=1` data.go.kr)는 이 쇼케이스 5명에 없음 → **실 API 그라운딩은 기상/해양 공공API(P2·P3)로 이동**. Problem Definition 부제도 "소상공인 여신·사후관리" 중심.

## 3. 실 LLM은 어디에 (권고)

**F1(오태림 기업여신)에 실 LLM을 꽂는다.** 텍스트 생성(기술신용 근거 패키징·품의 초안·회신 초안)이 LLM 가치가 눈에 보이는 지점이고, **Zero-PII 티어 라우팅**(분류·추출=로컬/저비용, 판단초안만 프런티어)이 원가·적법성 서사와 직결되며, **검증 에이전트가 자기 오류를 잡는 장면**이 신뢰의 절정. `/llm` 게이트웨이가 claude↔codex 폴백을 라이브 라우팅(timeoutMs 조작으로 사다리→사람 큐 격상 실연). 실 API 근거는 F2·F3의 기상/해양 공공API로 물린다.

## 4. 도장깨기 빌드 순서 (의존성 기반)

> 각 라운드 = 커밋 + 시연 가능한 슬라이스. 라운드 끝마다 데모 리허설 가능.

- **Gate 0 (선행·개발 게이트)**: **`/llm` 게이트웨이 JB_project2 이식** — 현재 예선 `02_제품`(handleLlm·llm-runs.jsonl·liveLlmBlock)에만 있음. 제출 정본=JB_project2에서 실 LLM(F1)을 보이려면 이식 필수. + PR#1(`ccl-financial`) 머지.
- **Round 1 — 코어(최소 세트, "이게 진짜 도냐" 격파)**: **F1**(실 LLM 오태림) → **F2**(실 API 문서희) → **F6**(계기판). 
- **Round 2 — 차별성("왜 너희냐" 격파)**: **F7**(나선형/조직도) → **F8**(엔터퍼스트) → **F4**(피싱 윤복례).
- **Round 3 — 풀 쇼케이스**: **F3**(송민재 선제감지) → **F5**(임세빈 학생) → **F9**(119/격리).

**시간 리스크 폴백**: 예선 5분/결선 15분. Round 1(3흐름)이 최소 승부 세트. Round 2까지가 1등 타깃. Round 3은 부록/백업 슬라이드·시연영상으로.
