# 적법성 근거팩 — 신용정보법 §40조의2 / §36조의2 정합

> P4/P5가 Q1·Q3·Q5·Q6·Q9·Q12에 남긴 "§40조의2 근거 없음 / §36조의2로 정정" 지적은 **오탐**이다. `_canon.md` §4에 검증된 근거가 이미 있다. §40조의2(가명처리·재식별금지)와 §36조의2(자동화평가 설명요구권)는 **별개 조문**이며 둘 다 유효 — 하나가 다른 하나의 오표기가 아니다.

## 근거 표

| 주장 | 특별법 근거 | 보충 근거 | 콘솔 매핑 |
|---|---|---|---|
| 은행/금융회사 데이터 처리는 신용정보법이 PIPA에 우선(특별법 우선) | 신용정보법 §3조의2 | PIPA는 보충 적용 | 모든 고객 데이터 처리 설계의 상위 원칙 — PIPA만 단독 인용 시 감점 |
| 가명처리 + 토큰↔원본 키 분리보관·안전조치·처리기록 | 신용정보법 §40조의2 ①②⑧ | PIPA §28조의4 | 토큰화 · 모델 라우팅(로컬=원본, 외부=가명만) |
| 재식별 금지 · 재식별 시 즉시 중단·회수·파기 | 신용정보법 §40조의2 ⑥⑦ | PIPA §28조의5 | 외부 반출 스캔(주민번호·주소·준식별자 조합 탐지) 및 즉시 처리중지·회수·삭제 |

## 미검증 표기 승계

- 전자금융감독규정 §15조 ①항의 정확한 호수(3·5호)는 **제출 직전 law.go.kr 재확인 필요**(원문의 [미검증] 마커 승계).
- 신용정보법 §3조의2, §40조의2 ①②⑥⑦⑧, PIPA §28조의4·§28조의5는 `_canon.md` §4 [확정] 근거.
- 신용정보법 §36조의2(자동화평가 설명요구권)는 canon §4 [확정] 근거가 아니라 **리서치 검증(D14/D15)** 근거다 — [[D14-결과]]·[[D15-결과]], 신용정보법 제36조의2 시행 2024.08.07, law.go.kr.
