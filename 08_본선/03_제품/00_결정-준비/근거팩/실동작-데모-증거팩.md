# 실동작 데모 증거팩

스파이크 근거: `/tmp/jb_spike/feasibility.md` (격리 환경 진단, 2026-07-03). 대상 머신은 이 스파이크를 실행한 개발 머신이며, 데모 당일 실사용 머신(예: 이승보 PC)의 상태는 별도 확인 필요 — 이 문서 범위 밖.

## 지금 가능한 것

- **정적 MVP 로컬 서빙** [확정] — `02_제품/app/`을 Node(v22.17.1)/Python(3.13.2) 기본 내장 서버로 이미 서빙 중(`npm run dev`). 추가 설치 불필요.
- **로컬 LLM 서빙 엔진** [확정] — `llama-server`(llama.cpp, Homebrew) 설치됨, Metal GPU 가속 확인(`GPU name: MTL0`). 가중치만 얹으면 추론 가능한 상태.
- **공공 오픈API 도달성** [확정] — law.go.kr(국가법령정보), MOLIT 실거래가(apis.data.go.kr), ECOS(한국은행) 3개 엔드포인트 모두 네트워크 차단 없이 응답(HTTP 200/401/307, 정상적인 "인증 실패" 응답 — 즉 스펙 확인 가능). FISIS는 리다이렉트만 확인.
- **배포 인프라 장벽 없음** [확정] — CLI(vercel/netlify/docker) 미설치지만 웹 UI로 GitHub 리포 연결 배포 가능, 또는 `npm install -g vercel` 한 줄로 즉시 설치 가능.

## "진짜 라이브"로 만드는 체크리스트

순서·오너는 [미검증] — 팀 논의로 확정 필요, 아래는 의존관계 기준 제안 순서.

1. **공공데이터포털 키 발급** (MOLIT + ECOS) — 즉시 무료 가능, 최우선. 오너: [미검증]
2. **law.go.kr용 배포 서버 IP/도메인 확정** — 다른 축과 무관하게 일정이 걸리므로 최대한 빨리 시작. 오너: [미검증]
3. **로컬 LLM 가중치 다운로드** (EXAONE 3.5 7.8B GGUF 우선, llama-server로 서빙 테스트) — 엔진은 이미 있음. 오너: [미검증]
4. **API 응답 → 앱 연동 코드 작성** (`computeRiskDecision` 등 기존 함수 계약에 실데이터 소스 연결) — 1·2 완료 후. 오너: [미검증]
5. **로컬 모델 추론 → 에이전트 판단 파이프라인 연결** — 3 완료 후. 오너: [미검증]
6. **통합 리허설 + 폴백 리허설**(아래 폴백 플랜 포함, 최소 1회) — 4·5 완료 후, 데모 전.

## 데모 폴백 플랜

| 실패 시나리오 | 폴백 |
|---|---|
| 로컬 모델 응답 지연(추론 시간 초과) | 사전 녹화한 데모 영상 구간으로 즉시 전환 + 동일 입력에 대한 캐시된 응답을 화면에 표시(실시간 추론처럼 재생하되 "사전 캐시" 명시는 발표자 재량) |
| 공공 API 장애(타임아웃/키 만료/레이트리밋) | 사전 확보한 스냅샷 응답(JSON)으로 즉시 대체 — 앱 내 폴백 데이터 소스로 미리 내장 |
| `localStorage` 리셋(브라우저/시연 환경 초기화) | 시드 데이터 재주입 스크립트(기존 데모 시나리오 `?demo=sme` 등 URL 파라미터로 즉시 재현) 준비 |

세 폴백 모두 "발표자가 눈치채자마자 1클릭/1커맨드로 전환 가능"해야 함 — 구체 스크립트/버튼은 별도 구현 항목(이 문서 범위 밖, 실행 백로그로 이관 필요).

## 심사 5축 "4.2 실동작·검증가능성" 대응 문장

> 로컬 LLM 서빙 엔진(llama-server, GPU 가속)과 정적 MVP 서빙 환경은 이미 구동 가능한 상태이며, 법령·부동산 실거래가·통화금융 공공 오픈API 3개 엔드포인트는 네트워크 도달 및 스펙 확인이 완료되어 인증키 발급만으로 실데이터 연동이 가능합니다. 남은 절차(모델 가중치 다운로드, 무료 인증키 발급)는 기술적 불확실성이 아닌 실행 대기 항목이며, 데모 안정성을 위해 로컬 모델 지연·공공 API 장애·상태 초기화 각각에 대한 폴백 경로를 사전 설계했습니다.

과장 금지 원칙에 따라 "완전히 라이브로 동작 중"이라는 표현은 사용하지 않음 — 현재 상태는 "엔진/도달성 확인, 실행 대기"이며 발표 시점까지 체크리스트 완료 여부에 따라 문장 갱신 필요.
