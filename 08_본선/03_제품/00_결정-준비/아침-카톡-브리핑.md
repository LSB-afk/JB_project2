회의록 3종(7/1~7/3) 인물별로 다시 정리했고, 우리가 아직 안 정한 질문 12개를 뽑아서 각각 후보안·데이터근거·추천안·심사위원 반박까지 다 붙여놨어요. 오늘 오전에 우리가 실제로 "골라야" 하는 것 위주로 정리합니다. 개발시간은 크게 안 잡아도 되고, 핵심 1개만 실제로 돌아가면 나머진 더미로 가도 된다는 방향이에요.

■ 밤새 AI가 근거까지 보강해둔 것 (거의 확정, 검토만):
- ROI 숫자: RM 116석 기준 연 7.66억원 절감(보수 0.83억~공격 31.42억). D16 실무 baseline(RM 월130건·심사의견서 30분→10초·시급 8.25만원)으로 바텀업 계산, 숫자 재검증까지 끝냄. → "5천만원? 우린 연 수억을 아낀다"로 말할 수 있음. (단 절감 실현율은 파일럿 실측이 남음 — 발표에선 "가정" 명시)
- 적법성: 신용정보법 §40조의2(가명처리·토큰 분리보관·재식별금지)+전자금융감독규정 망분리+금융위 MLS 로드맵으로 "외부 LLM 쓰되 원본 PII는 로컬"이 규제상 정당하다는 근거팩 완성. (§36조의2=AI 판단 설명요구권은 별개 근거로 병기)
- 실동작: 로컬모델 엔진(llama-server)·공공 오픈API(실거래가/법령/한국은행) 도달성 실측 완료. 무료 인증키 발급 + 로컬모델 가중치만 받으면 진짜로 돌아감.

■ 오늘 우리가 골라야 할 것 (추천안 — 확정 아님):
- Q1 도메인: 여신 1핵심(안전빵 실동작) + 전세사기·보이스피싱 2확장(뾰족한 사례). ※ 7/3 회의의 "3개 동급"안과 프레임이 달라서 이건 꼭 합의해야 함.
- Q3 에이전트 수: 14개 다 보여주기 vs 5코어+9스킬로 재분류. "적으면 안 된다"는 우리 방침과 충돌하니 이건 팀이 명시적으로 정해야 함.
- Q5 DB연동: 하이브리드 — 지금은 정적, 운영 전환 시 같은 4함수계약을 은행 read-only API로 교체("지금은 가짜지만 구조는 진짜와 똑같다").
- Q6 보안: 4중방어 라우팅 게이트 + 전세 외부조회는 비-PII 파라미터만.
- Q7 로컬모델: 로컬(Gemma/EXAONE/Qwen)이 원본 PII 처리, 외부 Claude/Codex는 비식별 문안만.
- Q9 스토리: RM 업무량 오프닝 → 개인 데모 케이스 마무리. (심사 5축엔 "스토리" 항목이 없으니 이걸 ②고객가치/④완성도 중 뭐로 걸지 정하기)
- Q11 이름: JB LocalGuard AX + 슬로건 "원본 PII는 로컬에, 결정은 승인체계 안에" (상표검색 전 잠정)
- Q12 승인 UX: Evidence-First Risk Gate(기본 + 위험별 마찰 + L4 차단)
- Q2/Q4/Q8/Q10: 링크 참고. 급한 건 위쪽.

■ 제일 급한 것: 7/4 오전에 "전세 vs 보이스피싱 중 뭘 실제로 돌릴지" 하나만 잠그면 Q1·Q6·Q7·Q8·Q9가 줄줄이 풀립니다.

■ 너는 이거:

## 1. 한 줄 요약
회의록 3종 인물별 재작성 + 결정 현황 종합(확정 10·가안 9·미결 12) + 미결 질문 Q1~Q12 후보·데이터근거·500자 프롬프트·심사위원 반박 + **2차 보강(적법성·ROI·실동작 3축 weak→improved, ROI 숫자 검증)**까지 완료. 결정은 오전 팀 몫.

## 2. 밤새 AI가 보강한 3축 (근거팩)
- **[[ROI-근거팩]]** — RM 116석 기준 연 **7.66억원** 절감(보수 0.83억~공격 31.42억), D16 baseline 바텀업·재검증 완료. 만능산술("0.018% 방어") 제거하고 방향성 단서로만. 실현율은 [추정](파일럿 필요). 우리캐피탈 수치는 [미검증]이라 산식서 제외.
- **[[적법성-근거팩]]** — 신용정보법 §40조의2 ①②⑥⑦⑧(특별법 우선)+전자금융감독규정 §15 망분리+금융위 MLS 로드맵(2024-08-13). §36조의2(설명요구권)은 D14/D15 별개 근거로 병기. 망분리 항·호 번호는 [미검증](제출 전 law.go.kr 재확인).
- **[[실동작-데모-증거팩]]** — 엔진·API 도달성 실측(n=1, 개발머신). 남은 건 무료키 발급 + 로컬모델 가중치. 데모 실패 폴백 플랜 포함. ※ "도달성 확인"이지 아직 "라이브 조회 성공"은 아님(정직 표기).

## 3. 지금 골라야 할 것 (질문별 추천 + 왜)
- **[[Q1-도메인 개수·조합]]** 여신 1핵심+전세·피싱 2확장 → 프레임 충돌(3개 동급 vs 위계) 재확인 필요.
- **[[Q2-RM 직책 포괄 범위]]** RM=사용자 페르소나, 제품="여신 리스크 운영 콘솔"로 스코프 명확화.
- **[[Q3-에이전트 수·규모·구성]]** 5코어+9스킬 재분류 or 14 유지 — "적으면 안 됨" 방침과 충돌, 팀 결정 필수.
- **[[Q4-5천만원 페인포인트·ROI]]** → [[ROI-근거팩]] 반영. 연 7.66억 절감 메인.
- **[[Q5-은행 DB 연결 방식]]** 하이브리드(정적→같은 함수계약을 read-only API로 승격).
- **[[Q6-보안 연결 아키텍처]]** 4중방어 라우팅 + 비-PII 외부조회.
- **[[Q7-로컬모델+API+OAuth 구동]]** 로컬=원본 PII, 외부=비식별.
- **[[Q8-배포 + 도메인 URL]]** 공개 정적 URL + 현장 로컬모델 하이브리드(도메인 [미검증]이면 발표서 제외).
- **[[Q9-스토리·공감(체스판)]]** RM 업무량 오프닝→데모 케이스. 심사축 매핑 결정 필요.
- **[[Q10-리더보드/순위 산정 기준]]** 105점 점수제 대비 설계 + 안내문 재확인.
- **[[Q11-시스템 이름·브랜딩]]** JB LocalGuard AX(상표검색 전 잠정).
- **[[Q12-위험 업무 승인 UX 구체 스펙]]** Evidence-First Risk Gate.
