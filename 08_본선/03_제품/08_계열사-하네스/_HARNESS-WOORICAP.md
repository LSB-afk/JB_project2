# JB우리캐피탈 도메인 하네스 (Domain Harness) — 계열사 정본 시스템 마스터 설계

> ⚠️ 대외비 — 팀 내부 설계 자료.
> **핵심 원칙 — 도메인 정본화(Domain-canon-by-default)**: *콘솔에 들어가는 계열사 사실(조직·업무·상품)은 전부 이 하네스에서 출발한다.* [[_HARNESS-SYSTEM|메인 운영 하네스]]가 "우리 팀의 작업"을 놓치지 않고 기록하듯, 이 하네스는 "타겟 계열사의 현실"을 놓치지 않고 정본화한다.
> 제품 근거: [[제품-정의]] §6/30 회의 — **타겟 = JB 2계열사(은행 1 + 비은행 JB우리캐피탈)**, 각 계열사 RM 에이전트가 여신·여신 사후관리·보이스피싱·전세사기 페인포인트 처리, **조직도 시각화 = 메인 UI**.

## 0. 두 종류의 하네스 (구분)

| 하네스 | 위치 | 다루는 것 | 용도 |
| --- | --- | --- | --- |
| **메인 운영 하네스** | `08_본선/_system/` | 우리 팀의 메모리·기록·협업 | 대회 운영·재현성 |
| **계열사 도메인 하네스(이 폴더)** | `08_본선/03_제품/07_계열사-하네스/` | 타겟 계열사의 조직·업무·상품·요구 | 콘솔 시드·조직도 UI·시나리오의 SSOT |

→ 콘솔(fork)의 조직도 렌더·케이스 시드·에이전트 배치가 계열사 현실과 어긋나면 **이 폴더가 이긴다**. 새 사실은 여기 먼저 쌓고 콘솔에 반영한다.

## 2. 기록 대상 & 스키마 (무엇을 정본으로 쌓는가)

1. **사실(Fact)** → 사이트·공시·보도자료에서 확인된 것. 반드시 `출처 URL + 접속일` 병기.
2. **해석(Interpretation)** → 여전업 도메인 지식으로 보완한 서술. 본문에 `(해석)` 표기 — 사실과 절대 섞지 않는다.
3. **매핑(Mapping)** → 계열사 현실 ↔ 콘솔 산출물(캐논 14 에이전트·케이스 시드·조직도 렌더) 연결. 변경 시 [[JB-콘솔-프로토타입-스펙-가안]]과 동기.
4. **수치 정합** → 재무 수치는 [[jb-group-business-facts]](예선 리서치 정본)와 충돌 금지. 충돌 발견 시 decision-log에 기록 후 정정.

## 3. 자동화·규약 (메인 하네스 규약 상속)

- **파일 생성/수정 후**: `canon-moc-sync` 실행(§AGENTS 4-A) — MOC 링크·frontmatter 정합.
- **의미 있는 변경**: `04_증빙/01_핵심로그/`에 append (Capture-by-default).
- **콘솔 반영 게이트**: 이 하네스의 조직·에이전트 배치 변경이 fork 시드(`jb-reseed-agents.ts`·`ui/src/data/`)에 영향을 주면 [[JB-콘솔-프로토타입-스펙-가안]] 갱신을 동반한다.
- **외부 공개 금지**: 사이트 인용은 분석 목적의 내부 자료. 제출물 인용 시 [[_canon]] 형식(출처·접속일)으로 승격 후 사용.
