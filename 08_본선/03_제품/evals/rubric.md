# evals/rubric — 심사기준 → 내부 지표 변환

## 목적

심사표를 "잘 썼다"가 아니라 **"측정했고 검증했다"**로 방어하기 위한 표다. 각 심사 기준을 재현 가능한 내부 지표(Internal Metric)로 바꾸고, 지표마다 증거·현재값·목표값을 못박는다. 절대값 KPI는 보장 문장이 아니라 `사전 정의된 평가 범위에서 관측치`로 표기한다. 〔D13: KPI는 분모·범위와 함께 말한다〕

## 근거등급(Evidence Grade) 정의

지표마다 아래 등급을 붙인다. **릴리스 목표 = E2+, 데모에서 실제로 보여줄 수 있으면 E4.**

| 등급 | 의미 |
|---|---|
| E0 | 주장만 있음, 근거 없음 |
| E1 | 외부 문헌/선례 인용 근거 |
| E2 | 우리 설계·규칙 문서로 정의됨(rules/, docs/) |
| E3 | 코드 계약·정적 검증(`verify_static.py`, `node --check`)으로 존재 확인 |
| E4 | e2e/실행으로 런타임 관측(Playwright, 실데이터 연동) |
| E5 | 외부 파일럿·감사 실측 |

⚠️ 아래 표의 `Current`는 2026-07-04 기준 스냅샷이며, 데모 준비 진행에 따라 갱신된다. 미확정 값은 [미검증]/[추정]으로 표기한다.
