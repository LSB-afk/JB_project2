# evals/failure-modes — 실패 모드 카탈로그

## 목적

> "우리 제품은 안전합니다"는 심사에서 감점이다. **"이런 실패가 나면 이렇게 탐지하고 이렇게 닫는다"**가 방어다. 이 문서는 `Case → AgentRun → Agent → Skill → Evidence → Approval → Audit` 운영계약과 PII 4중 방어가 **깨질 때**를 역으로 나열해, 시스템 불변식(승인 게이트·감사원장·fail-closed)이 실제로 무엇을 막는지 증명한다.

핵심 관점(리서치 정합):
- **사람 승인만 붙인다고 안전해지지 않는다.** rubber-stamping·false block·kill switch 미연결·입력 오염은 승인 게이트 위에 별도 통제가 있어야 막힌다. 〔D15 [E2]〕
- **속도만 재면 거의 틀린다.** faithfulness·재작업률을 같이 재지 않으면 "빨라진 오답"을 성공으로 오독한다. 〔D8 [E2]〕
- **모델 성능 하나가 문제가 아니다.** thin-file·전세 데이터 오류·FDS drift·한국어 금융모델 공백이 동시에 있다 → 데이터 신뢰도 게이트로 자동판단을 막는다. 〔D14 [E2]〕
- **수치를 지어내면 발표에서 무너진다.** 단일소스·간접추정·가정결합 수치를 확정값처럼 쓰면 심사위원 반문 한 번에 신뢰가 깨진다. 〔적대검증-리포트〕

## 읽는 법

- **근거등급 E0~E5** — E0 미검증 가정 · E1 문헌 시사 · E2 문헌 다중근거 · E3 내부 설계·코드 근거 · E4 데모 실증 가능 · E5 운영 실측.
- **신뢰마커** — **[확정]** canon/코드 직접 · **[조건부]** 설계 제안 · **[미검증]** 근거 약함 · **[Open Question]** 팀 결정 대기.
- **필수 필드**(스키마): `FailureID · Trigger · BadOutcome · DetectionMethod · RequiredFallback · UserMessage · Owner`. 각 행에 근거(D-ref/AGT-ID)를 병기한다.
- **Owner**는 [[08_본선/AGENTS|AGENTS]] 14역할 중 1차 책임자. 고객 대상 최종 판단은 승인 직군(**RM·준법**)으로 라우팅. 〔키스톤-확정〕

## Fallback 원칙 (fail-closed)

> 불확실하면 **열지 말고 닫는다.** 아래 우선순위로 강등한다. 〔principles P-004·D15〕

1. **차단이 안전이면 차단**(FM-C1 PII 반출, FM-A2 자동종결) — 실행보다 보류.
2. **자동이 위험이면 수기 검토**(FM-B1 thin-file, FM-B2 전세 시차) — `needsReview`로 라우팅.
3. **과잉 차단은 최소범위·빠른 복구**(FM-A4) — 부분 제한 + 5영업일 SLA + 재산정.
4. **모든 강등·차단·비상정지는 감사 기록**(FM-A3) — 사유코드 + 해시체인 + forensic export.
5. **고객 대상 최종 행동은 승인 후만**(FM-A5) — 보호성 내부 보류는 승인 전 허용, 통지·실행·조건변경은 승인 후.

## 자가 점검 — 약근거·미검증·가정

- **[미검증]** 국내 금융권 approval-first console이 사고를 몇 % 줄였는지 공개 1차 자료 없음 → FM-A 계열은 "구조로 강제한다"까지만 주장, 정량 효과 주장 금지. 〔D15 주의·한계〕
- **[미검증]** 한국 금융 RM·여신·사후관리 직무의 직접 RCT 근거 희소 → FM-E1은 "속도만 재지 말라"는 방어 원칙이지 생산성 수치 근거 아님. 〔D8 주의·한계〕
- **[미검증]** 전세사기용 공개 대규모 라벨 데이터셋 강하게 확인 안 됨 → FM-B2·B3의 자동탐지 성능은 목표치이지 실측 아님. 〔D14 주의·한계〕
- **[미검증]** 계열사 총자산 충돌값·SOHO 여신 잔액·과징금 10%·시행일 → FM-E3·E4에 따라 본문 확정값 사용 금지, 웹 재검증 대상. 〔적대검증-리포트〕
- **[조건부]** kill switch·사고보고 SLA(FM-D2·D3)·이상거래 보호성 보류 범위(FM-A5)는 설계 제안 — 구현·규제 재확인 필요.
- **[Open Question]** 승인 직군 2개(RM·준법) 유지 vs 확대 시 FM-A1 이중승인 라우팅 대상 재정의 필요. 〔키스톤-확정 팀 결정〕
- **[Open Question]** kill switch 물리 배선 범위(어떤 채널·배치·API까지)는 아키텍처 확정 후 FM-D1·D2에 반영.
