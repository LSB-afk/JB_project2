# IA·화면 맵

> [확정] JB LocalGuard OS 콘솔의 정보구조(IA) 명세. **원칙: 콘솔은 이미 빌드되어 있다 — 이 문서는 그 구조를 기술한다(반대 아님).** SSOT: [[JB-콘솔-프로토타입-스펙-가안]] §목표 IA·§시그니처 페이지. 에이전트 로스터·운영계약: [[_canon|canon]] §2·§8. 계열사 사실: [[_HARNESS-WOORICAP|우리캐피탈 하네스]].

---

## 1. 정보구조 개요

JB LocalGuard OS 콘솔은 3개 그룹, 16개 화면(콘솔 스펙 원문 표기 "15페이지" — 각주 1 참고)으로 구성된 **3열 셸**이다. 2026-06-26 초안이 상정했던 "다크 3열(org-rail 48px+sidebar 280px+workbench+속성패널)·8px 그리드"는 2026-07-02 콘솔 재설계에서 폐기됐다 — **라이트 테마가 기본**이며(OS가 다크여도 라이트 고정, 다크 토글 자체는 유지), 화면 그룹도 "케이스/에이전트 대시보드" 2분류가 아니라 **운영업무·AI조직·시스템 3그룹**으로 재편됐다.

- **운영업무** (6화면) — RM이 매일 여는 화면: 대시보드·알림함·케이스·승인·실행이력·전세보호.
- **AI조직** (4화면) — 에이전트 자체를 관리·감독: 에이전트·조직도·자동화·목표.
- **시스템** (6화면) — 거버넌스·비용·플랫폼 설정: 스킬·플러그인·활동·비용·PII거버넌스·설정.

> 각주 1: 콘솔 스펙 원문은 그룹 헤더에 "우리 예선 15페이지"라 적었지만 실제 표는 16행이다(운영업무6+AI조직4+시스템6=16). 원문 자체의 오차이며, 이 문서는 임의로 한쪽을 지워 맞추지 않고 16화면 전체를 인벤토리에 싣는다. 표기 정정은 콘솔 스펙 쪽에서 처리할 사안(이 문서 범위 밖).

## 3. 시그니처 4페이지 (P3, 실동작)

콘솔 스펙 §시그니처 페이지 그대로. 4개 중 2개(전세보호·PII거버넌스)는 **나비게이션 목록 화면**(S-06, S-15)이고, 나머지 2개(리스크 L0~L4·감사체인)는 독립 nav 항목이 아니라 **케이스 상세(S-03)와 승인(S-04)에서 열리는 context-panel 드릴인 뷰**다 — 3열 셸에서 "사람은 눈만"으로 근거를 확인하는 자리이므로 별도 nav 슬롯을 쓰지 않는 것이 의도적 설계다.

1. **전세 Shield**(S-06, `/jeonse`) — 진단 폼 → 전세가율/자산노출/주거비/종합위험 계산 → 체크리스트·특약초안·은행연계(HUG) 생성. 고위험(근저당·신탁) 판정 시 자동으로 **L3 pending**(준법 검토 필요) 라우팅. `runJeonseDiagnosis` 미러. 오너: 전세위험 관리 리드.
2. **PII 거버넌스**(S-15, `/pii`) — 등급제(원본/가명/토큰화)·토큰화 전/후 비교·모델 라우팅(내부 계산은 로컬, 추론만 외부 LLM)·반출 스캔(원본 PII 유출 시도 탐지)·근거 법령 칩(신용정보법 §40조의2, 개인정보보호법 §28조의4/5). `modules.js` governance 미러. 오너: 준법 검토 에이전트.
3. **리스크 L0~L4 뷰**(케이스 상세 드릴인, `/risk` 또는 `/cases/:id`) — `computeRiskDecision` 신호 분해(actionType별 5신호×가중치) + score→L레벨 승인 매트릭스 시각화. 케이스 카드의 리스크 배지 클릭 시 context-panel에 열림.
4. **감사 체인 뷰**(케이스 상세/실행이력 드릴인, `/audit`) — `auditChainRecords` 해시체인(GENESIS→각 레코드 seq·actor·action·evidenceId·previousHash·hash) 무결성 시각화. 승인(S-04) 완료 직후 자동으로 이 뷰가 열려 "방금 승인이 체인에 기록됐다"를 보여주는 것이 표준 흐름.

## 4. 내비게이션 모델

콘솔 셸은 index.html 마운트 포인트 기준 4개 DOM 리전이 "3열"로 지각되도록 배치된다(`org-rail`, `nav-list`, `#page-content`, `#context-panel`):

```
[ org-rail ]  [ nav-list | page-content ]  [ context-panel ]
   계열사        3그룹 아코디언 + 현재 화면       드릴인 상세
   전환 아이콘    (S-01~S-16 중 1개)             (리스크/감사체인/
                                                근거 등, 옵션)
```

- **org-rail**(최좌측, 아이콘 rail) — 계열사 전환. 확정 스코프 = **전북은행 + JB우리캐피탈**(canon §5 전체 5개 계열사 중 2개). 두 계열사를 활성 아이콘으로, 나머지 3개(광주은행·JB자산운용·JB인베스트먼트)는 비활성/준비중 배지로 노출하는 안을 제안한다 — 심사기준 5.3(계열사 확장 가능성)을 시각적으로 증명하면서도 데모 스코프는 2개로 좁게 유지하는 절충 (TBD, 디자인 제안이며 SSOT 확정 아님).
- **nav-list** — 3그룹(운영업무/AI조직/시스템) 아코디언, 그룹당 화면 목록. 기본 펼침 그룹 = 운영업무.
- **page-content** — 선택된 화면(S-01~S-16)의 실제 콘텐츠.
- **context-panel** — 케이스/승인/에이전트를 클릭했을 때 우측에서 슬라이드인. 리스크 L0~L4 분해, 감사체인, 근거(Evidence) 원문 등 "드릴인 전용" 정보가 여기 산다. 모션: 카드 진입 600ms(`--motion-card`), 메뉴류 280ms(`--motion-menu`) — [[08_본선/03_제품/03_ux/design-system|디자인 시스템]] §모션 참고.

## 5. 키보드 퍼스트 흐름

원칙: **"일은 에이전트, 사람은 눈만"** — RM은 타이핑이 아니라 확인·승인만 한다. 핵심 제스처는 **스페이스 2회로 진행**이다.

| 입력 | 동작 |
|---|---|
| `Space` (1회) | 포커스된 항목(케이스/승인 카드)을 펼쳐 근거를 확인 — context-panel 오픈 |
| `Space` (연속 2회) | 근거 확인 완료 → 다음 미검토 항목으로 스킵(리뷰 큐 전진) |
| `Enter` | 승인 |
| `Shift+Enter` | 반려/수정 요청 |
| `↑` / `↓` (또는 `k`/`j`) | 리스트 항목 이동 |
| `Esc` | context-panel 닫기 |

> `Space` 2회 진행 제스처는 [확정](콘솔 UX 원칙). 나머지 개별 키(Enter/Shift+Enter/↑↓/Esc)는 이 문서의 디자인 제안(TBD) — 구현 시 콘솔 실측 키맵으로 대체.
