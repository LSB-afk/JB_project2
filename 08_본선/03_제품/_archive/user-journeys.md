# 사용자 여정

> [확정] 3개 핵심 데모 여정. 전 여정은 운영계약 `Case → AgentRun → Agent → Skill → Evidence → Approval → Audit`([[_canon|canon]] §8) 7단계에 그대로 매핑된다. 고객 대상 행동은 승인 전 자동 실행 금지. 확정 스코프(계열사=전북은행+JB우리캐피탈, 도메인=여신·전세·피싱): [[_canon|canon]] §0·§1. 콘솔 실측 화면: [[ia-screen-map|IA·화면 맵]](미확정 초안, `_archive/`).

> **실동작 범위 [조건부]**: 여신(J1, 히어로)은 **더미 시나리오**로 시연(스크립트된 타임라인). 전세(J2)·피싱(J3) 중 **1개만 실동작**(사용자 입력에 실시간 반응하는 진단/판정) — 7/4 최종 확정 예정. 이 문서는 확정 전까지 두 여정을 동등한 상세도로 기술하고, 실동작 여부만 [조건부]로 표시한다.

## 0. 운영계약 7단계 — 공통 프레임

| 단계 | 이 문서의 표기 | 콘솔 화면 |
|---|---|---|
| Case | 케이스 생성 | S-03 케이스 |
| AgentRun | 에이전트 판단(실행) | S-05 실행이력 |
| Agent | (판단 주체, 표 내 명시) | S-07 에이전트 |
| Skill | 행동 초안 | S-11 스킬 |
| Evidence | 검증(근거 연결) | 리스크 L0~L4 드릴인 |
| Approval | 사람 승인(L레벨) | S-04 승인 |
| Audit | 감사체인 기록 | 감사체인 드릴인 |

## Journey 1 — 히어로: 전주 중앙로 카페 대출심사 [확정, 더미 시연]

**시나리오**: 전북은행 개인사업자 고객, 전주 중앙로 카페. 케이스 `JBG-104`, riskScore **88**, 상태 `Approval Pending`(urgent). 노출: 운전자금 1.8억·카드매출 둔화. 핵심 pain: 금리 부담+매출 둔화. 데모 진입: `?demo=sme`. KPI: Triage 50%↓.

| 단계 | 내용 |
|---|---|
| **신호 유입** | 카드매출 둔화 패턴을 위험신호 조기감지 에이전트(L0)가 야간 자동화 스캔(S-09)에서 탐지 |
| **Case** | JBG-104 케이스 오픈, 상태 `New` → `Approval Pending` |
| **AgentRun / Agent** | **상환위험 분류 에이전트**(히어로 케이스 owner)가 `computeRiskDecision` 실행. actionType=`customerNotice`(소상공인), 신호 5종×가중치: 상환스트레스32·정책금융22·서류18·근거16·안내12 → 합산 score **88** → **L3 밴드(80~89)** 진입 — 이 케이스는 상환위험 분류 에이전트의 평소 운영 레벨(L1/2)을 넘어 **준법 검토 에이전트**가 개입하는 고위험 경로로 자동 승격 |
| **Skill(행동 초안)** | 정책금융 매칭 에이전트가 대환대출/정책자금 후보 매칭 초안, RM 보좌 에이전트가 고객 안내 문구 초안 |
| **Evidence(검증)** | 카드매출 데이터, 신용정보 스냅샷, 정책금융 요건 매핑을 근거로 연결(근거연결률 100% KPI) — 리스크 L0~L4 드릴인에서 5개 신호별 기여도 확인 가능 |
| **Approval(사람 승인)** | 1차 **준법 최종 승인자**(L3, score 80~89대 자동 라우팅) 검토 → 2차 **RM 최종 승인자** 최종 승인. 승인 게이트가 고객 발송 전 최종 차단선 |
| **Audit** | `auditChainRecords` 해시체인: GENESIS→위험신호 감지→케이스 오픈→상환위험 판정→정책금융 초안→준법 검토→RM 승인→고객 안내 발송, 각 seq에 evidenceId·previousHash·hash |

**IA 경로**: S-01 대시보드(고위험 클러스터에서 진입) → S-03 케이스(JBG-104 상세) → 리스크 L0~L4 드릴인 → S-04 승인 → 감사체인 드릴인.

**키보드 퍼스트 예시**: 대시보드에서 `Space` 2회로 최우선 고위험 케이스로 스킵 → 케이스 상세에서 `Space` 1회로 5개 신호 근거 확인 → `Enter`로 승인.

## Journey 2 — 전세사기 대응 [조건부: 실동작 여부 7/4 확정]

**시나리오**: 전세위험 관리 리드가 조율하는 전세보호 라인. 배경 통계: 전세사기 피해자 누적 인정 **39,121건**(국토부 2026-06-09), HUG 사고액 2024년 4.49조(역대 최고)→2025년 1.24조. 데모 진입: `?demo=jeonse`. KPI: Jeonse safe-contract 체크 100%.

| 단계 | 내용 |
|---|---|
| **신호 유입** | 위험신호 조기감지 에이전트가 고전세가율/깡통전세 패턴 후보를 탐지, 전세위험 관리 리드에게 라우팅 |
| **Case** | 전세 케이스 오픈, actionType=`contract`(전세) |
| **AgentRun / Agent** | 전세가율 분석 에이전트(전세가율 계산) · 등기 권리 분석 에이전트(권리관계·근저당·신탁 확인) · 임차인 손실위험 에이전트(손실 가능액 산정)가 병렬 실행. `computeRiskDecision` 신호 5종×가중치: 전세가율34·권리24·임차인노출18·보증14·은행10 |
| **자동 승격** | 등기 권리 분석에서 근저당·신탁 등 고위험 권리관계 발견 시 **자동으로 L3 pending**(준법 검토 필요) — 콘솔 스펙에 명시된 확정 동작 |
| **Skill(행동 초안)** | 계약 체크리스트 에이전트가 특약 초안 생성, 은행 연계 에이전트가 HUG 보증 연계 초안 |
| **Evidence(검증)** | 등기부등본 발췌, 전세가율 계산 근거, HUG 보증 요건 매핑 |
| **Approval(사람 승인)** | **준법 최종 승인자**(L3) 검토·승인 → 승인 게이트 |
| **Audit** | 해시체인 기록: GENESIS→신호 탐지→케이스 오픈→전세가율/권리/손실위험 3판정→특약 초안→준법 승인 |

**IA 경로**: S-06 전세보호(진단 폼·시그니처 페이지) → S-03 케이스 상세 → S-04 승인 → 감사체인 드릴인.

**[조건부] 표시**: 이 여정이 "1개 실동작" 후보로 선정되면 S-06의 진단 폼이 사용자 입력에 실시간 반응(전세가율 게이지 즉시 계산)한다. 선정되지 않으면 J1과 동일하게 스크립트된 더미 타임라인으로 시연.

## Journey 3 — 보이스피싱 이상거래 차단 [조건부: 실동작 여부 7/4 확정]

**시나리오**: 이상거래 탐지·차단 에이전트가 1차 자동 차단하는 고위험 사기 라인. 배경 통계: 2025년 1~11월 피해액 **1조 1,330억(+56.1% YoY)**, 건당 평균 5,248만원(1~11월), 50대 이상 53%(2025 Q1 기준). 데모 진입: `?demo=phishing`. KPI: 승인 100%.

| 단계 | 내용 |
|---|---|
| **신호 유입** | 이상거래 탐지·차단 에이전트가 실시간 거래 패턴(외부 URL 유도·비정상 이체 등)을 감지 |
| **Case** | fraud 케이스 오픈, actionType=`fraud` |
| **AgentRun / Agent** | 이상거래 탐지·차단 에이전트(L4)가 `computeRiskDecision` 실행. 신호 5종×가중치: 외부URL34·차단필요28·AI악용22·준법16 |
| **자동 차단(L4)** | score가 90~100(L4 밴드)이면 **사람 승인 이전에 거래/접촉을 시스템이 선차단** — 이 에이전트의 canon 소속 레벨이 L4("차단·상위검토")이므로, 승인 게이트는 차단 이후의 **상위검토**로 작동한다(사전 승인이 아니라 사후 확인) |
| **Skill(행동 초안)** | 준법 검토 에이전트가 신고/법적 조치 초안, RM 보좌 에이전트가 피해 고지 안내 초안 |
| **Evidence(검증)** | 이상거래 패턴 로그, 외부 URL 스캔 결과, AI 악용 시그널 근거 |
| **Approval(사람 승인)** | 승인 게이트가 자동 차단을 유지한 채 **RM 최종 승인자·준법 최종 승인자**가 상위검토(L4) — 승인 100% KPI는 이 상위검토 단계가 빠짐없이 수행됨을 뜻한다 |
| **Audit** | 해시체인 기록: GENESIS→이상거래 감지→즉시 차단→케이스 오픈→상위검토→고지 발송 |

**IA 경로**: S-02 알림함(긴급 큐, L4 최우선) → S-03 케이스 상세(FraudShield 워크플로) → S-04 승인(상위검토) → S-13 활동/감사체인 드릴인.

**[조건부] 표시**: J2와 동일한 미확정 조건 — 전세/피싱 중 하나만 실동작.
