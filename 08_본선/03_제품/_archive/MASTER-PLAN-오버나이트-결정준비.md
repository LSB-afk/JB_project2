# MASTER-PLAN — 오버나이트 결정준비 배치 (2026-07-03 밤 → 07-04 아침 보고)

> **왜 지금 이걸 하나.** 정의서(스펙)를 바로 쓰기 전에, 우리는 아직 **확정을 안 했다**. 7/1·7/2·7/3 회의를 바탕으로 **바텀업으로 "결정거리 + 근거 있는 후보"를 정리**해야, 아침에 팀이 보고 *"오 이거·이거·이거 우리가 골라서 하면 되겠다, 그 이유는 이러이러하다"* 를 즉시 판단할 수 있다. 모든 후보는 **논리적·데이터 기반**이어야 하고, **깐깐한 금융 심사위원이 납득**할 수준이어야 한다.

## 0. 산출물 (아침에 이 파일들만 보면 됨)
전부 `08_본선/03_제품/01_결정-준비/` 아래.
1. **[[결정-현황-종합]]** — ✅확정 / 🔶가안 / ❓미결 (오너·마감·의존). 3개 회의 + 리서치 + 확정 스냅샷 통합.
2. **`질문/Q01~Q0N-*.md`** — 열린 질문별 **후보 A/B/C + 데이터 근거 + 추천안(뾰족한 이유) + 리스크 + ~500자 리서치 프롬프트 + 심사위원 반박 대비**.
3. **[[놓친것-질문과요청]]** — 사용자/팀이 놓친 것 → 질문·요청으로 정리.
4. **`회의록-정리본/*`** — 3개 회의 원문을 **인물별 줄글**로 재작성(리액션=긍정/부정만, 의미 왜곡 0, 원문 완전보존).
5. **`viz/*.md`(Mermaid)** — 사람이 캡처→카톡으로 바로 이해하는 보드(결정현황·인물별 할일·제품 한눈).
6. **[[아침-카톡-브리핑]]** — 팀원별 "너는 이거" + 실제 자료 링크/경로.

## 1. 실행 구조 — 병렬 에이전트 + 루프 (자율 오버나이트)
Workflow 하네스로 **백그라운드 자율 완주**(내 Opus 토큰 최소화). 실무는 **Sonnet 병렬 + Codex($200, 헤드리스 `codex exec`)**, 최종 종합·카톡만 Opus.

| 페이즈 | 방식 | 산출 |
|---|---|---|
| P1 회의록 처리 | 3 병렬(회의당 1) | 인물별 줄글 재작성 + 결정/가안/미결/액션 구조 추출 |
| P2 결정 종합 | 1(고티어) | 확정/가안/미결 마스터 + **최종 질문 리스트** |
| P3 질문별 리서치 | N 병렬(Codex 우선) | 기존 리서치(D1~D31) 마이닝 → 후보 A/B/C + 근거 + 500자 프롬프트 |
| P4 심사위원 반박 | N 병렬(파이프라인) | 깐깐한 금융심사 렌즈로 추천안 **반박 시도** → 유지/약함/기각 + 보강점 |
| P5 완결성 비평 | 1~2 루프 | "빠진 질문·미검증 후보·심사위원이 더 물을 것" → 미결 목록 보강 |
| P6 시각화 | 2 병렬 | 인간친화 Mermaid 보드 |
| P7 종합·카톡 | Opus(내가) | 추천 종합 + 아침 카톡 + HANDOFF 갱신 |

**루프/자기점검**: P4 반박 + P5 비평이 "문제 없나·개선점 필요한가"를 강제. 추천안이 심사 렌즈에서 기각/약함이면 보강점을 후보 문서에 회수.

## 2. 열린 질문 (P3 착수 셋 — P2/P5에서 추가 가능)
- **Q1 도메인 개수·조합** — 현 가안 3~4개(여신·전세사기·보이스피싱[+?]). 더 뾰족한 조합? 우리가 못 본 도메인? **작게 잡지 않되 뾰족한 이유 필수**. (↔ Q2)
- **Q2 RM 직책 포괄 범위** — 콘솔이 RM을 완전히 포괄하나? RM 업무 실범위·타 직무와의 차이 → 도메인 가감.
- **Q3 에이전트 수·규모·구성** — 적으면 안 됨(금융=하나 놓치면 큰일). 모듈식(담당자별 상이). 이름=한 가지 잘하는 직원. **에이전트 신뢰도 평가(모델평가式)**.
- **Q4 5천만원 페인포인트·ROI** — 실제 손실/비용 정량 + 에이전트 도입 시 비용절감·생산성향상 정량. 심사 납득 데이터.
- **Q5 은행 DB 연결 방식** — 전산 입력정보를 에이전트가 함께 읽고 **고객 메모리/근거문서로 활용·작성**. 연결 가능성 시연법 + 빠른 실현가능성 테스트.
- **Q6 보안 연결 아키텍처** — 스킬·플러그인·MCP·외부DB·정책·금융뉴스·법령·실거래가·전세가·국토부 오픈API·공개신뢰데이터 **안전 연결 + 시연**.
- **Q7 로컬모델 + API + OAuth 구동** — 콘솔에서 실제 구동 + 결과물. 모델 택.
- **Q8 배포 + 도메인 URL** — 시험 배포·제출용 URL 방법.
- **Q9 스토리·공감(체스판)** — 우리 스토리·고객 스토리 + 발표 전략·전술.

## 3. 검증 렌즈 (모든 추천안이 통과해야)
깐깐한 금융 심사위원: ① 페인포인트가 **실재·정량**인가(데이터 출처) ② 적법성(신용정보법·PIPA·망분리) ③ 신뢰성·정확성(하나 놓치면?) ④ ROI가 **5천만원급**인가 ⑤ 실동작·배포 증명 ⑥ 확장성. 근거는 기존 리서치(D5a 법조문·D14/15 신뢰성·D30 시장·D31 모수·D+a ROI)에서 인용.

## 4. 가드레일
- **원문 완전보존**: 재작성본은 파생물, Downloads 원본·기존 회의록 미삭제/미덮어쓰기.
- **의미 왜곡 0**: STT 재작성은 인물별 병합·리액션 축약만, 해석·창작 금지.
- **SSOT**: `_canon`·콘솔 스펙 우선, 미검증 수치 `_canon` 반영 금지, 신뢰마커 `[확정]/[조건부]`. Perplexity 미사용(기존 리서치 마이닝).
- **병렬 세션 공존**: 이 배치 산출은 전부 신규 폴더 `00_결정-준비/`에 격리, 남의 파일 미접촉, 커밋은 내 것만.
- **승인 게이트**: 외부 제출/공개/고객 대상 = 사람. **직장인 자격 문제는 이번엔 무시**(사용자 지시).
