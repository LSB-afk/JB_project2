# 00 Source Log — 입력 자료·출처·신뢰도 대장

> **역할**: 이 문서 세트(DDBM·Harness·SDD 노멀폼)가 근거로 삼는 **입력 자료의 추적성 대장(SSOT of sources)**이다. 여기서 등록된 자료만 하위 문서의 근거로 인용한다. 신규 자료는 먼저 여기에 등록하고 신뢰도·출처를 붙인 뒤 인용한다.
> **최상위 출처 규율**: **대회 안내 정본([[심사기준]])과 심사기준 5축·105점**이 모든 판단의 1순위 출처다. 안내문과 충돌하는 내부 문서는 안내문이 이긴다.
> **SSOT 규율**: 사실·수치·에이전트명·법령 인용은 지어내지 않는다. 아래 백본 SSOT의 표현을 그대로 재사용한다(근거등급 E0~E5 부착).

---

## 0. 이 문서 읽는 법 (신뢰도·근거등급 범례)

**신뢰도(Reliability)** — 자료 자체의 출처 성격:
| 등급 | 의미 |
|---|---|
| **A** | 공식 정본·1차 출처(대회 안내문, 실제 코드, 법령 원문) |
| **B** | 내부 확정 SSOT(팀 합의·백본 문서·근거팩) |
| **C** | 내부 작업 산출물(회의록·설계 메모·미확정 권고) |
| **D** | 외부 리서치 요약(2차 자료·전문가 Q&A 생성, 검증 필요) |

**근거등급(Evidence Level, 하위 문서에서 주장에 부착)** — [[_문서생성-스킬-DDBM-Harness-SDD]] §Evidence Levels:
`E0 가설 · E1 내부관찰/회의 · E2 외부자료 · E3 사용자검증 · E4 작동검증(데모) · E5 운영검증`. 목표 E2+, 데모가능=E4.

> ⚠️ 신뢰도(A~D)는 **자료의 출처 성격**, 근거등급(E0~E5)은 **개별 주장의 입증 강도**다. 둘은 다르다. 예: 리서치 요약(신뢰도 D)이 인용한 공공 통계는 하위 문서에서 원문 확인 후 E2로 승격 가능.
