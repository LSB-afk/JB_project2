# AI 협업·맥락관리 프로토콜

> 여러 AI 세션(Claude·Codex, 동시 3~4개)이 **같은 볼트를 Syncthing으로 실시간 공유**하며 충돌 없이 일하기 위한 룰북. 실제로 겪은 사고에서 뽑은 규칙이다.
> 관계: [[AGENTS]](역할·결정모델·승인게이트) + [[collaboration-rules]](브랜치·로그·핸드오프6블록·비밀유지) 위에 **병렬 세션 공존 + 맥락 관리**를 얹는다. 중복 규칙은 여기 다시 안 쓴다.

---

## 1. 세션 생애주기 (BOOT → WORK → HANDOFF)

| 단계 | 무엇 | 도구 |
|---|---|---|
| **BOOT** | 진입점·현재상태·미결·자동트리거·승인게이트 로드 | `node 08_본선/_system/skills/session-boot/boot.mjs` + 오늘 [[_daily-INDEX]] 노트 |
| **WORK** | 작업. 의미있는 행동은 즉시 로그(Capture-by-default) | `04_증빙/01_핵심로그/` + 데일리 노트 append |
| **HANDOFF** | 종료 시 6블록 + 데일리 "열린 핸드오프" 갱신 | [[collaboration-rules]] §4 + [[_daily-INDEX]] |

**하루 첫 세션**은 [[_daily-template]]로 그날 데일리 노트를 만든다. 이후 세션은 거기에 "🟢 활성 세션 / 🔀 열린 핸드오프"를 append 해 **서로 뭘 하는지 공지**한다.

---

## 2. 병렬 세션 공존 (⚠️ 사고에서 나온 규칙)

1. **커밋은 내 파일만 명시 스테이징.** `git add -A`·`git commit -a` 금지. `git add <구체경로>`로만. 다른 세션이 진행 중인 파일을 휩쓸어 커밋하면 그쪽 작업이 깨진다.
2. **미커밋(진행중) 파일은 그 세션 소유** — 건드리거나 커밋하지 않는다. 특히 `_canon.md`(SSOT reconcile), `05_제출/리서치-딥프롬프트/_결과/_종합/`. 남의 반쯤 된 SSOT를 커밋하면 정본이 오염된다.
3. **경로는 프로젝트 루트 앵커.** 스크립트·훅은 `CLAUDE_PROJECT_DIR` 기준으로 쓴다. `cwd`(하위 폴더일 수 있음) 기준으로 쓰면 중첩 `08_본선/…/08_본선/` 트리가 생겨 공유 데이터가 쪼개진다. 그런 트리가 보이면 상대경로 사고 신호(gitignore 백스톱이 잡지만 원인 스크립트를 고쳐라).
4. **Syncthing = 실시간 양방향.** 내가 저장한 순간 다른 기기·세션에 반영된다. 큰 파일 이어쓰기 전 최신 상태 확인. `*.sync-conflict-*`는 동시편집 충돌본(gitignore).
5. **파일 rename·이동 금지** — 옵시디언 인바운드 `[[ ]]`가 끊긴다. 잘못된 위치의 파일은 옮기지 말고 정본 위치에 다시 만들고 링크를 고친다.
6. **서브에이전트 병렬**은 서로 다른 파일에 쓸 때만. 같은 파일 동시 편집은 마지막 쓰기가 이긴다.

---

## 3. 맥락 관리 (context)

1. **SSOT 규율** — 사실·수치·에이전트명은 정본에서만. 예선=`_canon.md`, 본선=`_system/memory/` + `_체계/` 대회 정본. 충돌 시 정본이 이긴다. **발명 금지**.
2. **신뢰마커** — 수치·주장엔 `[확정]`(정본 확인) / `[조건부]`(기준기간·근거등급 병기) / `[격리]`(단정 금지, 가정/범위/부록). 확정 아닌 값을 본문에 단정하지 않는다.
3. **미결정은 옵션 병기** — 사람이 정할 것(제품 결정 등)을 AI가 발명하지 않는다. 옵션 병기 + "승인 대기". 게이트 현황은 [[다음-작업-분해]].
4. **링크·도달성 측정은 `canon-moc-sync`** (`grep` 금지 — NFC/이스케이프 파이프 함정). 파일 생성·삭제·이름변경 후 반드시 실행: `node 08_본선/_system/skills/canon-moc-sync/sync.mjs`.
5. **긴 세션·컴팩션** — 요약 후에도 정본(데일리·session-log·PROGRESS)이 상태를 보존하므로 거기서 이어받는다. 기억에 의존하지 말고 기록을 읽는다.
6. **맵 파일은 손으로 안 고친다** — MOC·레지스트리는 스킬로만 갱신([[collaboration-rules]] §6).

---

## 4. 승인 게이트 (요약, 상세는 [[AGENTS]])

- **자율**: 회수본 인용·문서 작성·신뢰마커·검증·내부 커밋/fork 푸시.
- **제안→승인**: 구조 변경·정본 수치 변경·새 표준/설정.
- **사람만(자율 금지)**: 외부 제출·공개·고객 대상, `_canon` 히어로 프레이밍 직접수정, 미결 제품결정 택1, 결제 민감정보 기록.

---

## 5. 세션 체크리스트

**시작**: ☐ session-boot 실행 ☐ 오늘 데일리 노트 확인(없으면 생성) ☐ 다른 세션 "활성/핸드오프" 읽기 ☐ `git status`로 진행중 파일 파악

**종료**: ☐ 내 파일만 스테이징·커밋 ☐ session-log 6블록 ☐ 데일리 "열린 핸드오프" 갱신 ☐ canon-moc-sync 통과(링크 변경 시) ☐ 남의 미커밋 파일 안 건드렸는지 확인
