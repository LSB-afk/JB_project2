# meeting-intake

> 회의 STT → **정리본 1개**(추적, 발췌 줄글+구조화 요약 통합) + FULL.txt(gitignore, 완전 원본 아카이브) + 인덱스·메모리·거버넌스. **AI는 이 절차대로 일관 실행**한다(팀원은 몰라도 기록은 누적). 검증된 실사례: 2026-06-27 브리핑·준비, 2026-06-30 아이디어 회의.
> ⚠️ **2026-07-04 "발췌→정리본 승격" 정리**: 예전엔 정리본(추적)·발췌(gitignore) 2파일 구조였으나, gitignore 폴더는 AI가 다른 실행 환경(git-only clone 등)에서 못 읽어 정작 풍부한 내용이 AI에 안 보이는 문제가 있어 **1파일 구조로 통합**했다. 아래 절차는 이 통합 구조 기준.

## 언제
- 사용자가 회의 녹취 `.txt`(STT 전문) 경로를 공유하거나 "회의록 옮겨/정리/기록"을 요청할 때.

## 절차 (순서대로)

### 1. 원문 정독 + 대회 분리
- STT 전문을 끝까지 읽는다(길면 페이지 분할). **JB 대회 관련만** 발췌 대상. 타 대회(데이콘 스마트팜·노각장·성대·삼성챌린지 등)·사적 잡담은 제외(혼선·범위오염 방지).
- 화자명은 STT로 뭉개지니 역할 매핑이 확실할 때만 실명, 아니면 `참석자N` 병기.

### 2. FULL.txt 아카이브 — gitignore (대외비·PII)
- 경로: `08_본선/04_증빙/04_회의록/_원문/원문-YYYY-MM-DD-주제-FULL.txt` — 원본 STT를 무편집으로 그대로 복사(완전 축약없는 verbatim 아카이브).
- `_원문/`은 `.gitignore`에 이미 등록됨(Syncthing 로컬 공유만). 원본 `.txt`는 별도로도 보존(미삭제).

### 3. 회의록 정리본 — 추적 (발췌 줄글 + 구조화 요약 통합 1파일)
- 경로: `08_본선/04_증빙/04_회의록/회의록-YYYY-MM-DD-주제.md`
- frontmatter: `area/evidence · type/meeting · status/active · up:"[[_회의록-INDEX]]"`, aliases.
- 실명 컨벤션 유지(기존 정리본이 실명 사용: 김주용·이승보·김민주 등). 단 ⚠️ 공개 fork 푸시 시 실명·내부전략 노출 → 사용자에게 플래그.
- ⚠️ **결제 민감정보(카드번호·비밀번호)는 어떤 파일에도 기록 금지** → `[결제정보 redacted]`. 상금 분배 등 금액 합의는 "합의 완료" 수준으로.
- 구조(예시=`회의록-2026-07-04-UX업무보드흐름.md`): `## 참석/맥락` → `## 핵심 요약(3줄)` → **`## 발췌 — 주제별 서술(프롬프트 인풋용 줄글)`**(핑퐁 금지, 소제목+타임스탬프 범위+서술형 문단, FULL.txt 내용을 축약 없이 담되 핑퐁 아닌 줄글로) → `---` → **`## 공유·결정(구조화 요약)`**(번호 리스트) → `## 일정` → `## 액션 아이템`(체크박스) → `## 미확정·플래그` → `## 연결`([[_회의록-INDEX]]·관련 플레이북, FULL.txt는 플레인 경로로 언급).
- ⚠️ STT의 수치(매출·인원 등)는 **미검증** → 회의록엔 적되 `_canon` 반영 금지(SSOT 오염 방지).

### 4. 인덱스·MOC
- `08_본선/04_증빙/04_회의록/_회의록-INDEX.md` 회의 목록 표에 행 추가(일시·주제·정리본·FULL.txt). "회의 → 수정 필요 파일" 표 갱신.
- 회의록은 INDEX가 부모로 링크 → INDEX는 `_04_증빙_MOC` 자식이라 도달성 자동 확보(별도 MOC 행 불필요).

### 5. 수정 필요 파일 도출 + 반영
- 회의 결정으로 바뀌어야 할 파일을 도출해 INDEX 표에 ✅반영/🔶제안으로. 확정 항목은 즉시 반영(예: 제품정의·플레이북·레지스트리), 사람 판단 필요분은 🔶로 남김.

### 6. 메모리 + 거버넌스
- 메모리: 인-볼트 `08_본선/_system/memory/context/*`(제품정의·현황 등) + 개인 메모리(`~/.claude/projects/.../memory/`) 갱신·인덱스(MEMORY.md) 1행.
- 거버넌스: `decision-log.md`(회의 결정 1항목), `PROGRESS.md`, `본선-마스터-플레이북`(회의 반영 절).

### 7. 정합성 검증 (필수)
- 새 파일 frontmatter+up 확인 후 **[[canon-moc-sync]]** 실행 → `[4/5]` 죽은링크 0, `[5/5]` 도달성(루트→회의록 도달) 확인. 누락 시 `--apply` 또는 부모 링크 보강.

## 가드레일
- JB 한정 · 결제 민감정보 레닥션 · 미검증 수치 _canon 금지 · FULL.txt gitignore · 외부 공개/푸시는 사람 승인.
