# 리서치 점-점 연결 · 야간 종합 파이프라인 (Implementation Plan)

> **에이전트 워커용**: 이 플랜은 야간 자율 실행용. 각 Phase 완료(코덱스 배치 알림) 시 다음 Phase를 잇는다. 산출은 모두 로컬 누적(푸시 금지), canon은 **제안만**(자동수정 금지).

**Goal:** 30+개 독립 리서치 결과를 코덱스(gpt-5.5 high) 병렬로 **점-점 연결 → 교차 인사이트 → 본선 논증 척추**로 합성해 활용도를 극대화한다.

**Architecture:** 3-Phase 파이프라인. A(클러스터별 연결노트, 6병렬) → B(인사이트맵·모순통합) → C(본선 논증척추: 인사이트→심사5기준·제품정의·시연). Codex가 합성 콘텐츠를, Claude가 vault 배선(MOC·링크·canon-moc-sync)·거버넌스·메모리를 담당.

**Tech Stack:** `codex exec -m gpt-5.5 -s read-only --output-last-message`(읽기전용·1파일 산출), pandoc(이미 회수), Claude(오케스트레이션·canon-moc-sync·기록).

## Global Constraints (전 Phase 공통)
- **푸시 금지.** 팀 repo·fork 모두 야간 푸시 안 함. 로컬 누적 → 아침 사용자 검토.
- **canon 자동수정 금지.** 모순·신규사실은 `_종합/_canon-갱신후보.md`에 **제안만**. SSOT 변경은 사람 승인(외부/공개/제출 게이트 준용).
- **출처 보존.** 인사이트마다 근거 결과파일(`[[D..-결과]]`)·원 수치 인용. 새 수치 창작 금지.
- **휴머나이즈.** 산출은 사람 말투(짧은 능동문, AI티 금지) — 팀이 바로 읽게.
- **Capture-by-default.** Phase 전이마다 프롬프트로그·session-log·텔레메트리에 1줄. 새 파일은 `up:`로 부모연결(canon-moc-sync [5/5]).
- **비용.** 코덱스=망상궤도 공용 Pro(토큰 공유). 배치 총 ~8잡으로 bound.

## Phase A — 클러스터별 점-점 연결노트 (Codex 6병렬, read-only)
각 잡 = 한 클러스터의 `_결과/*.md`를 읽고 **연결노트 1파일** 산출. 산출 구조(고정):
`## 연결된 발견`(어느 결과의 무엇이 서로 맞물리나) · `## 1차 인사이트`(결합으로 새로 보이는 것 3~5) · `## 모순·텐션`(+reconcile 제안) · `## 제품·심사 매핑`(5기준 중 무엇을 입증) · `## 시연 한 컷` · `## canon 갱신 후보`.

- [ ] **A-1 제품 척추**: D20·D22·D23(gpt55pro+gemini)·B1 → "model-agnostic 거버넌드 OS + 경제성 증명(NPV·자본구제)".
- [ ] **A-2 인텔리전스**: D21·D7a·D7b·D11·D17 → 통합 고객지능·정책룰화·시뮬레이션(합법범위=D17/D5a 분리).
- [ ] **A-3 규제·신뢰**: D5a·D5b·D10·D12·D15 → PII 비반출·승인우선의 법·기술 정합.
- [ ] **A-4 JB 적합·시장·ROI**: D3a·D3b·D3c·D3d·D3e·D3f·D+a·D16 → 왜 JB·왜 지금·바이어 ROI.
- [ ] **A-5 검증·운영**: D8·D13·D18·D19·D+b → 측정·현업신뢰·운영리스크·확장.
- [ ] **A-6 도메인·시스템**: D2·D6·D9·D14 → 도메인지식·코어뱅킹 정합·아키텍처·데이터품질.
- **수용기준**: 6파일 생성, 각 6섹션 채움, 결과파일 인용 존재, 모순 섹션 비어있지 않음(없으면 "없음+근거").
- **전이**: 6잡 완료 알림 → Claude가 품질 점검 → Phase B 착수.

## Phase B — 교차 인사이트 합성 + 모순 통합 (Codex 1, after A)
연결노트 6개를 읽고:
- [ ] `_인사이트맵.md` — **교차 인사이트(insight-from-insight) Top 5~8**(여러 클러스터를 관통하는 명제) + **점-점 매트릭스**(어느 도메인쌍이 서로 강화/충돌) + 각 인사이트의 근거 클러스터.
- [ ] `_canon-갱신후보.md` — 결과 간·_canon과의 **모순 목록 + 통합 제안**(예: D23 GPT 24.2억 vs Gemini 33.8억 reconcile). **제안만**.
- [ ] `_미해결갭.md` — 인사이트가 드러낸 빈 곳 → 차기 리서치 후보(D24~ 또는 흡수).
- **수용기준**: Top 인사이트마다 ≥2 클러스터 근거, 모순 reconcile 논리 명시, 갭 트리거 표기.

## Phase C — 본선 논증 척추 (Codex 1, after B)
- [ ] `_본선-논증척추.md` — 인사이트 → **심사 5기준**(주제/금융가치/Agent설계/MVP검증/혁신·운영) × **제품정의**(JB 2계열사·은행+JB우리캐피탈·RM에이전트, [[본선-제품정의-확정]]) × **시연 시나리오** 매핑. "이 인사이트로 이 점수를 이렇게 딴다"의 척추.
- **수용기준**: 5기준 전부 ≥1 인사이트로 뒷받침, 시연 컷과 연결, 제품정의 정합.
