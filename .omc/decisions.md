# OMC Decision Log - JB LocalGuard OS

이 문서는 이번 골 기능 실행에서 내린 주요 제품/기술 결정을 기록한다. Fabel 5 자문은 외부 도구 호출이 아니라, 심사 관점의 내부 advisory pass로 다섯 번 검토했다.

## 2026-06-13 Judging-Ready Pass

| 체크포인트 | Fabel 5 의견 | 반영 여부 | 결정 |
| --- | --- | --- | --- |
| 사용자 시나리오 우선순위 | 심사위원은 "처음부터 끝까지 실제로 되는가"를 먼저 본다. 전세, 보이스피싱, 소상공인 자금압박 3종을 골든 패스로 고정해야 한다. | 반영 | `?demo=jeonse`, `?demo=phishing`, `?demo=sme` 데모 모드를 만들고 각 모드에서 선택 케이스, 분석 결과, 다음 행동을 deterministic하게 시작하도록 했다. |
| UI/UX 개선 방향 | 한 화면에 모든 것을 압축하기보다, 실행자가 현재 가치와 다음 행동을 바로 읽어야 한다. | 반영 | 데모 코치마크, 점수 분해, 출처 칩, 승인 라우팅 문구를 추가해 "왜 이 결론인가"를 즉시 확인하게 했다. |
| 비용·통계 대시보드 설계 | 총액보다 "그래서 무엇을 먼저 처리할지"가 중요하다. | 반영 | 예상 비용, 절감 가능 위험, ROI와 함께 최우선 처리군을 문장으로 생성하는 비용 해석 로직을 추가했다. |
| 버그 수정 우선순위 | 저장/감사/승인처럼 심사 시연에서 끊기면 치명적인 흐름을 우선 검증해야 한다. | 반영 | 감사 해시 체인 검증, JSON 내보내기, schemaVersion 저장, Playwright 데모 검증을 우선순위로 구현했다. |

## 고정 원칙

- Vanilla JS/CSS 구조를 유지한다.
- 데모 데이터와 실제/공공 출처 기반 설명을 구분한다.
- AI Agent 판단은 사람 승인 레벨과 감사 로그로 연결한다.
- 향후 실제 API 연동 시에도 `computeRiskDecision`, `auditChainRecords`, `buildDashboardData`의 경계를 유지한다.
